## 1. Cleona Chat คืออะไร?

### Messenger ของคุณ ข้อมูลของคุณ

Cleona Chat เป็นแอปส่งข้อความ (Messenger) ที่ทำงานได้โดยไม่ต้องพึ่งพาเซิร์ฟเวอร์กลางเลย ข้อความของคุณจะถูกส่งตรงจากอุปกรณ์ของคุณไปยังอุปกรณ์ของคู่สนทนา -- โดยไม่ผ่านสำนักงานใหญ่ของบริษัทใด ไม่ผ่านคลาวด์ ไม่ผ่านศูนย์ข้อมูลใด ๆ ไม่มีบริษัทใดสามารถอ่าน จัดเก็บ หรือส่งต่อข้อความของคุณได้ เพราะไม่มีบริษัทใดอยู่ตรงกลางเลย

### ไม่ต้องมีบัญชี ไม่ต้องมีเบอร์โทรศัพท์

ในการใช้งาน Cleona คุณไม่จำเป็นต้องมีเบอร์โทรศัพท์หรือที่อยู่อีเมลเพื่อลงทะเบียน ตัวตนของคุณประกอบด้วยคู่กุญแจเข้ารหัสที่ถูกสร้างขึ้นโดยอัตโนมัติบนอุปกรณ์ของคุณเมื่อเปิดใช้งานครั้งแรก นั่นหมายความว่า ไม่มีใครสามารถค้นหาตัวคุณผ่านเบอร์โทรศัพท์หรือที่อยู่อีเมลได้ เว้นแต่คุณจะเปิดเผยข้อมูลติดต่อของคุณเอง

### การเข้ารหัสที่พร้อมรับมืออนาคต

Cleona ใช้การเข้ารหัสแบบต้านทานควอนตัม (Post-Quantum) ซึ่งหมายความว่า แม้แต่คอมพิวเตอร์ควอนตัมในอนาคตก็ไม่สามารถถอดรหัสข้อความของคุณได้ คุณไม่จำเป็นต้องเข้าใจรายละเอียดด้านเทคนิค -- สิ่งสำคัญคือการสื่อสารของคุณได้รับการปกป้องอย่างดีที่สุดเท่าที่เทคโนโลยีปัจจุบันจะทำได้

### Cleona ทำงานได้อย่างไรโดยไม่มีเซิร์ฟเวอร์?

ลองนึกภาพว่าคุณและผู้ติดต่อของคุณรวมกันเป็นเครือข่ายหนึ่งเดียว อุปกรณ์ทุกเครื่องช่วยกันส่งต่อข้อความ หากคู่สนทนาของคุณออนไลน์อยู่ในขณะนั้น ข้อความจะถูกส่งตรงถึงเขาทันที หากคู่สนทนาออฟไลน์ ผู้ติดต่อร่วมกันจะเก็บข้อความนั้นไว้ชั่วคราวและส่งต่อให้เมื่อผู้รับกลับมาออนไลน์อีกครั้ง ดังนั้นผู้ติดต่อของคุณจึงเป็นทั้งผู้รู้จักและเครือข่ายของคุณในเวลาเดียวกัน

### แพลตฟอร์ม

Cleona พร้อมใช้งานบน Android, iOS, macOS, Linux และ Windows

## 2. เริ่มต้นใช้งาน

### ติดตั้งแอป

**Android:**
1. ดาวน์โหลดไฟล์ APK จากเว็บไซต์ Cleona หรือจาก GitHub Releases
2. เปิดไฟล์บนโทรศัพท์ของคุณ หากจำเป็น ให้อนุญาตการติดตั้งจากแหล่งที่ไม่รู้จัก (Android จะถามคุณโดยอัตโนมัติ)
3. แตะ "ติดตั้ง" และรอจนกว่าการติดตั้งจะเสร็จสมบูรณ์

**iOS:**
1. เปิดลิงก์คำเชิญ TestFlight บน iPhone ของคุณ
2. แตะ "ติดตั้ง" TestFlight เป็นช่องทางอย่างเป็นทางการของ Apple สำหรับเผยแพร่แอปเวอร์ชันเบต้า
3. หลังการติดตั้งเสร็จสิ้น คุณจะพบ Cleona บนหน้าจอหลักของคุณ

**macOS:**
1. ดาวน์โหลดไฟล์ DMG จากเว็บไซต์ Cleona หรือจาก GitHub Releases
2. เปิดไฟล์ DMG แล้วลาก Cleona ไปยังโฟลเดอร์ Programme ของคุณ
3. เมื่อเปิดใช้งานครั้งแรก macOS อาจถามว่าคุณต้องการเปิดแอปจากนักพัฒนาที่ได้รับการยืนยันตัวตนหรือไม่ -- ให้ยืนยันการเปิด

**Linux (Ubuntu/Debian):**
1. ดาวน์โหลดไฟล์ .deb จากเว็บไซต์ Cleona หรือจาก GitHub Releases
2. ติดตั้งด้วยการดับเบิลคลิก หรือในเทอร์มินัลด้วยคำสั่ง: `sudo dpkg -i cleona-chat_VERSION_amd64.deb`
3. เปิด Cleona ผ่านเมนูแอปพลิเคชัน หรือในเทอร์มินัลด้วยคำสั่ง `cleona-chat`

**Linux (Fedora/openSUSE):**
1. ดาวน์โหลดไฟล์ .rpm จากเว็บไซต์ Cleona หรือจาก GitHub Releases
2. ติดตั้งด้วยคำสั่ง: `sudo dnf install cleona-chat-VERSION.x86_64.rpm`
3. เปิด Cleona ผ่านเมนูแอปพลิเคชัน หรือในเทอร์มินัลด้วยคำสั่ง `cleona-chat`

**Linux (ทุกดิสทริบิวชัน -- AppImage):**
1. ดาวน์โหลดไฟล์ .AppImage จากเว็บไซต์ Cleona หรือจาก GitHub Releases
2. ทำให้ไฟล์สามารถเรียกใช้งานได้: คลิกขวา เลือกคุณสมบัติ (Properties) ตั้งค่าให้เรียกใช้งานได้ หรือในเทอร์มินัลด้วยคำสั่ง: `chmod +x cleona-chat-VERSION-x86_64.AppImage`
3. เริ่มการทำงานด้วยการดับเบิลคลิก หรือในเทอร์มินัลด้วยคำสั่ง: `./cleona-chat-VERSION-x86_64.AppImage`

**Windows:**
1. ดาวน์โหลดตัวติดตั้งจากเว็บไซต์ Cleona หรือจาก GitHub Releases
2. เรียกใช้ไฟล์ติดตั้งและทำตามขั้นตอนที่ปรากฏ
3. เปิด Cleona ผ่านเมนูเริ่มต้น (Startmenü) หรือทางลัดบนเดสก์ท็อป

### สร้างตัวตน

เมื่อเปิดใช้งานครั้งแรก Cleona จะสร้างตัวตนใหม่ให้คุณโดยอัตโนมัติ คุณสามารถตั้งชื่อที่แสดงของคุณเองได้ -- ซึ่งเป็นชื่อที่ผู้ติดต่อของคุณจะมองเห็น ชื่อนี้สามารถเปลี่ยนแปลงได้ทุกเมื่อ

### จดบันทึก Seed-Phrase -- สิ่งสำคัญที่สุด

หลังจากสร้างตัวตนของคุณเสร็จแล้ว Cleona จะแสดงคำจำนวน 24 คำให้คุณเห็น นี่คือ **Seed-Phrase** ของคุณ -- กุญแจกู้คืนข้อมูลส่วนตัวของคุณ

**จดคำ 24 คำนี้ลงบนกระดาษและเก็บรักษาไว้อย่างปลอดภัย**

เหตุใดเรื่องนี้จึงสำคัญมาก?

- หากโทรศัพท์ของคุณเสีย สูญหาย หรือถูกขโมย คุณสามารถใช้คำ 24 คำนี้กู้คืนตัวตนทั้งหมดของคุณบนอุปกรณ์เครื่องใหม่ได้
- หากไม่มี Seed-Phrase จะไม่มีทางย้อนกลับได้เลย ไม่มีปุ่ม "ลืมรหัสผ่าน" และไม่มีฝ่ายสนับสนุนที่จะคืนบัญชีให้คุณได้ -- เพราะไม่มีบัญชีอยู่บนเซิร์ฟเวอร์เลยตั้งแต่แรก
- ห้ามเปิดเผย Seed-Phrase ให้ผู้อื่นทราบเด็ดขาด ผู้ที่รู้คำเหล่านี้สามารถแอบอ้างเป็นคุณได้

คุณสามารถดู Seed-Phrase อีกครั้งภายหลังได้ในเมนูการตั้งค่า หัวข้อ "ความปลอดภัย" หากต้องการอ่านซ้ำ

### เพิ่มผู้ติดต่อคนแรก

หากต้องการสนทนากับใครสักคน คุณต้องเพิ่มบุคคลนั้นเป็นผู้ติดต่อก่อน ซึ่งมีหลายวิธีในการทำเช่นนี้ -- ทั้งหมดจะอธิบายไว้ในหัวข้อถัดไป

## 3. รายชื่อผู้ติดต่อ

### สแกน QR-Code (แนะนำ)

วิธีที่ง่ายที่สุดในการเพิ่มผู้ติดต่อ:

1. คู่สนทนาของคุณเปิดหน้ารายละเอียดตัวตนของเขา (แตะที่ชื่อของตัวเองในแถบด้านบน) และแสดง QR-Code ให้คุณดู
2. คุณแตะปุ่มเครื่องหมายบวก แล้วเลือก "สแกน QR-Code"
3. เล็งโทรศัพท์ของคุณไปที่ QR-Code ของคู่สนทนา
4. คำขอเป็นผู้ติดต่อจะถูกส่งโดยอัตโนมัติ เมื่อคู่สนทนายอมรับคำขอแล้ว ทั้งสองฝ่ายก็จะสามารถส่งข้อความหากันได้

หากคุณพบกันตัวต่อตัว QR-Code คือวิธีที่ปลอดภัยที่สุด เพราะคุณจะรู้แน่ชัดว่ากำลังแลกเปลี่ยนข้อมูลติดต่อกับใคร

### NFC (นำโทรศัพท์มาแตะกัน)

หากอุปกรณ์ทั้งสองเครื่องรองรับ NFC:

1. เปิดฟังก์ชันเพิ่มผู้ติดต่อบนอุปกรณ์ทั้งสองเครื่อง
2. นำโทรศัพท์ทั้งสองเครื่องมาแตะด้านหลังชนกัน
3. ข้อมูลผู้ติดต่อจะถูกแลกเปลี่ยนโดยอัตโนมัติ

NFC ให้ความปลอดภัยสูงเช่นเดียวกับ QR-Code เพราะการแลกเปลี่ยนข้อมูลจะสำเร็จได้ก็ต่อเมื่อทั้งสองฝ่ายอยู่ใกล้กันในทางกายภาพเท่านั้น

### แชร์ลิงก์ (cleona:// URI)

คุณยังสามารถส่งลิงก์ผู้ติดต่อของคุณผ่านทางอีเมล SMS หรือแอปส่งข้อความอื่น ๆ ได้:

1. เปิดหน้ารายละเอียดตัวตนของคุณ
2. คัดลอกลิงก์ cleona:// ของคุณ
3. ส่งลิงก์ให้กับบุคคลที่ต้องการเพิ่มคุณเป็นผู้ติดต่อ
4. อีกฝ่ายเปิดลิงก์ หรือวางลิงก์นั้นในกล่องโต้ตอบเพิ่มผู้ติดต่อ

โปรดทราบ: วิธีนี้ต้องอาศัยความไว้วางใจว่าลิงก์จะไม่ถูกดัดแปลงระหว่างการส่ง สำหรับผู้ติดต่อที่มีความละเอียดอ่อนเป็นพิเศษ เราแนะนำให้ใช้ QR-Code หรือ NFC แทน

### ยอมรับคำขอเป็นผู้ติดต่อ

เมื่อมีใครส่งคำขอเป็นผู้ติดต่อถึงคุณ คำขอนั้นจะปรากฏในกล่องข้อความเข้า (แท็บสุดท้ายในแถบด้านล่าง) ที่นั่นคุณสามารถ:

- **ยอมรับ** -- บุคคลนั้นจะถูกเพิ่มเข้าในรายชื่อผู้ติดต่อของคุณ
- **ปฏิเสธ** -- คำขอจะถูกยกเลิก
- **บล็อก** -- บุคคลนั้นจะไม่สามารถส่งคำขอถึงคุณได้อีก

### ระดับการยืนยันตัวตน

Cleona จะแสดงให้คุณเห็นว่าตัวตนของผู้ติดต่อได้รับการยืนยันมั่นคงเพียงใด:

| ระดับ | ความหมาย |
|-------|-----------|
| ไม่ทราบ | คุณได้รับเพียง Node-ID หรือลิงก์เท่านั้น |
| เคยเห็น | การแลกเปลี่ยนกุญแจสำเร็จแล้ว ทั้งสองฝ่ายสามารถสื่อสารแบบเข้ารหัสกันได้ |
| ยืนยันแล้ว | ทั้งสองฝ่ายพบกันด้วยตัวเองและยืนยันตัวตนผ่าน QR-Code หรือ NFC แล้ว |
| ไว้วางใจ | คุณได้ทำเครื่องหมายผู้ติดต่อรายนี้ว่าน่าเชื่อถือโดยชัดแจ้งแล้ว |

ยิ่งระดับสูงเท่าไร คุณก็ยิ่งมั่นใจได้มากขึ้นเท่านั้นว่ากำลังสนทนาอยู่กับบุคคลที่ถูกต้องจริง ๆ

## 4. ข้อความ

### ส่งและรับข้อความตัวอักษร

เพียงพิมพ์ข้อความของคุณลงในช่องป้อนข้อความด้านล่างแล้วกด Enter หรือปุ่มส่ง ข้อความของคุณจะถูกเข้ารหัสโดยอัตโนมัติก่อนที่จะออกจากอุปกรณ์ของคุณ

ข้อความที่เข้ามาจะปรากฏในประวัติการสนทนา เครื่องหมายถูกจะแสดงให้คุณเห็นว่าข้อความของคุณถูกส่งถึงผู้รับแล้วหรือยัง

### ส่งรูปภาพ วิดีโอ และไฟล์

คุณมีหลายวิธีในการทำเช่นนี้:

- **ไอคอนคลิปหนีบกระดาษ** ในช่องป้อนข้อความ: แตะที่ไอคอนนี้เพื่อเลือกไฟล์ รูปภาพ หรือวิดีโอจากแกลเลอรีหรือระบบไฟล์ของคุณ
- **ลากแล้ววาง** (เดสก์ท็อป): เพียงลากไฟล์เข้าไปในหน้าต่างแชท
- **วางจากคลิปบอร์ด** (เดสก์ท็อป): คัดลอกรูปภาพแล้ววางลงในแชท

ไฟล์ขนาดเล็ก (ต่ำกว่า 256 KB) จะถูกส่งไปพร้อมกับข้อความโดยตรง ส่วนไฟล์ขนาดใหญ่กว่านั้นจะถูกส่งด้วยกระบวนการสองขั้นตอน: อันดับแรกไฟล์จะถูกประกาศแจ้งก่อน จากนั้นจึงส่งเป็นส่วน ๆ

### ข้อความเสียง

1. กดปุ่มไมโครโฟนในช่องป้อนข้อความค้างไว้
2. พูดข้อความของคุณ
3. ปล่อยปุ่มเพื่อส่งข้อความ

หากเปิดใช้งานการถอดเสียงเป็นข้อความ (ดูในการตั้งค่า) บนอุปกรณ์ของคุณ ข้อความเสียงของคุณจะถูกถอดเป็นข้อความตัวอักษรโดยอัตโนมัติ คู่สนทนาของคุณจะเห็นทั้งไฟล์เสียงต้นฉบับและข้อความที่ถอดเสียงแล้ว

### ตอบกลับข้อความ (อ้างอิงข้อความ)

หากต้องการตอบกลับข้อความใดข้อความหนึ่งโดยเฉพาะ:

1. เปิดเมนูสามจุดข้างข้อความนั้น
2. เลือก "ตอบกลับ"
3. แบนเนอร์แสดงข้อความที่ถูกอ้างอิงจะปรากฏขึ้นเหนือช่องป้อนข้อความ
4. พิมพ์คำตอบของคุณและส่งออกไป

ข้อความที่ถูกอ้างอิงจะแสดงอยู่ในคำตอบของคุณ ทำให้เห็นความเชื่อมโยงได้อย่างชัดเจน

### แก้ไขและลบข้อความ

- **แก้ไข:** เปิดเมนูสามจุดของข้อความ แล้วเลือก "แก้ไข" เปลี่ยนข้อความแล้วส่งใหม่อีกครั้ง คู่สนทนาของคุณจะเห็นว่าข้อความนี้ถูกแก้ไขแล้ว การแก้ไขทำได้ภายใน 15 นาทีหลังจากส่งข้อความ
- **ลบ:** เปิดเมนูสามจุดของข้อความ แล้วเลือก "ลบ" ข้อความจะถูกลบออกทั้งจากฝั่งคุณและฝั่งคู่สนทนา คุณสามารถลบข้อความของคุณเองได้ทุกเมื่อ -- ไม่มีข้อจำกัดด้านเวลาสำหรับการลบ

### ปฏิกิริยาอีโมจิ

แทนที่จะพิมพ์คำตอบ คุณสามารถตอบสนองต่อข้อความด้วยอีโมจิได้:

1. เปิดเมนูสามจุด หรือกดค้างที่ข้อความนั้น
2. เลือกอีโมจิจากตัวเลือกด่วน หรือเปิดตัวเลือกอีโมจิแบบเต็มรูปแบบ
3. ปฏิกิริยาของคุณจะปรากฏใต้ข้อความนั้น

### คัดลอกข้อความ

ผ่านเมนูสามจุดของข้อความ คุณสามารถคัดลอกข้อความตัวอักษรลงในคลิปบอร์ดได้

### ค้นหาข้อความ

ที่ขอบด้านบนของหน้าต่างแชท คุณจะพบฟังก์ชันค้นหา พิมพ์คำค้นหาลงไป แล้ว Cleona จะแสดงผลลัพธ์ที่พบทั้งหมดในแชทปัจจุบัน คุณสามารถใช้ปุ่มลูกศรเพื่อสลับไปมาระหว่างผลลัพธ์ที่พบได้

ในหน้าหลักยังมีตัวกรองค้นหาข้ามแท็บเพิ่มเติม ซึ่งช่วยให้คุณค้นหาคำในทุกบทสนทนาได้พร้อมกัน

### ตัวอย่างลิงก์

เมื่อคุณส่งลิงก์ Cleona จะสร้างตัวอย่าง (ชื่อเรื่อง คำอธิบาย ภาพตัวอย่าง) ให้โดยอัตโนมัติ ตัวอย่างนี้ถูกสร้างขึ้นบนอุปกรณ์ของคุณและส่งไปพร้อมกับข้อความ -- คู่สนทนาของคุณไม่จำเป็นต้องเชื่อมต่อกับเว็บไซต์ที่ลิงก์ไปถึงเลย

เมื่อคุณแตะที่ลิงก์ที่ได้รับ ระบบจะถามคุณว่าต้องการเปิดในเบราว์เซอร์ปกติ ในโหมดไม่ระบุตัวตน หรือไม่เปิดเลย

## 5. กลุ่ม

### สร้างกลุ่ม

1. สลับไปที่แท็บ "กลุ่ม"
2. แตะปุ่มเครื่องหมายบวก
3. ตั้งชื่อให้กลุ่ม
4. เลือกผู้ติดต่อที่คุณต้องการเชิญ
5. แตะ "สร้าง"

ผู้ติดต่อที่ได้รับเชิญจะได้รับการแจ้งเตือนและสามารถเข้าร่วมกลุ่มได้

### เชิญสมาชิก

แม้หลังจากสร้างกลุ่มแล้ว คุณก็ยังสามารถเชิญผู้ติดต่อเพิ่มเติมได้:

1. เปิดข้อมูลกลุ่ม (เมนูสามจุดในหน้าภาพรวมกลุ่ม หรือแถบด้านบนในหน้าแชทกลุ่ม)
2. แตะ "เชิญ"
3. เลือกผู้ติดต่อที่คุณต้องการเพิ่ม

### บทบาท

แต่ละกลุ่มมีบทบาทสามระดับ:

- **เจ้าของ (Owner):** มีสิทธิ์ควบคุมทั้งหมด สามารถเพิ่มและลบสมาชิก แต่งตั้งแอดมิน และจัดการกลุ่มได้ เจ้าของสามารถโอนสถานะของตนให้กับสมาชิกคนอื่นได้เช่นกัน
- **แอดมิน (Admin):** สามารถลบสมาชิกและช่วยจัดการกลุ่มได้
- **สมาชิก (Mitglied):** สามารถอ่านและเขียนข้อความได้

### ออกจากกลุ่ม

1. เปิดเมนูสามจุดในหน้าภาพรวมกลุ่ม
2. เลือก "ออกจากกลุ่ม"
3. ยืนยันการตัดสินใจของคุณ

เมื่อคุณออกจากกลุ่ม ข้อความที่คุณเคยส่งไว้จะยังคงปรากฏให้สมาชิกคนอื่น ๆ เห็นตามเดิม

## 6. ช่องสาธารณะ

### ช่องคืออะไร?

ช่องคือฟอรัมสนทนาสาธารณะภายในเครือข่าย Cleona ต่างจากกลุ่มตรงที่ทุกคนสามารถอ่านเนื้อหาได้โดยไม่ต้องได้รับเชิญ มีเพียงเจ้าของและแอดมินเท่านั้นที่สามารถโพสต์เนื้อหาได้ -- ผู้ติดตามมีสิทธิ์อ่านเพียงอย่างเดียว

### ค้นหาและเข้าร่วมช่อง

1. สลับไปที่แท็บ "ช่อง"
2. เปิดแท็บ "ค้นหา"
3. ค้นหาช่องที่มีอยู่ตามชื่อหรือหัวข้อ
4. แตะที่ช่องหนึ่ง แล้วแตะ "ติดตาม"

ช่องสามารถกรองตามภาษาได้ บางช่องถูกระบุว่า "ไม่เหมาะสำหรับเยาวชน" -- ช่องเหล่านี้จะปรากฏให้เห็นก็ต่อเมื่อคุณยืนยันในโปรไฟล์ของคุณแล้วว่ามีอายุเกิน 18 ปี

### สร้างช่องของคุณเอง

1. สลับไปที่แท็บ "ช่อง"
2. แตะปุ่มเครื่องหมายบวก
3. ตั้งชื่อช่อง (ต้องไม่ซ้ำกับช่องอื่นทั่วทั้งเครือข่าย)
4. เลือกภาษาและกำหนดว่าช่องนี้จะเป็นสาธารณะหรือส่วนตัว
5. เพิ่มเติม (ไม่บังคับ): ใส่คำอธิบายและรูปภาพ
6. แตะ "สร้าง"

สำหรับช่องสาธารณะ คุณสามารถกำหนดได้ว่าเนื้อหาจะถูกจัดประเภทเป็น "ไม่เหมาะสำหรับเยาวชน" หรือไม่

### รายงานเนื้อหา

หากคุณพบเนื้อหาที่ไม่เหมาะสมในช่องสาธารณะ คุณสามารถรายงานได้ Cleona ใช้ระบบตรวจสอบเนื้อหาแบบกระจายศูนย์: รายงานจะถูกพิจารณาโดยสมาชิกเครือข่ายที่ถูกสุ่มเลือกขึ้นมา (คล้ายกับ "คณะลูกขุน") หากพบว่ามีการละเมิดจริง ช่องนั้นจะได้รับคำเตือน หากมีการละเมิดซ้ำหลายครั้ง ช่องนั้นจะถูกลดอันดับในดัชนีค้นหาหรือถูกระงับการใช้งาน

### ช่องระบบ

Cleona มีช่องระบบในตัวสองช่อง:

- **Bug Log:** เมื่อ Cleona ตรวจพบข้อผิดพลาด ระบบจะถามคุณว่าต้องการส่งรายงานข้อผิดพลาดแบบไม่ระบุตัวตนหรือไม่ รายงานเหล่านี้จะถูกส่งไปยังช่อง Bug Log ซึ่งสมาชิกในชุมชนสามารถเข้าดูได้ จะไม่มีการส่งข้อมูลส่วนบุคคลใด ๆ -- มีเพียงคำอธิบายข้อผิดพลาดทางเทคนิคเท่านั้น คุณยังสามารถส่งรายงานบันทึกด้วยตนเองได้เช่นกัน (พร้อมกล่องโต้ตอบแสดงตัวอย่างและการยินยอมอย่างชัดแจ้ง)
- **Feature Requests:** ที่นี่ผู้ใช้สามารถส่งข้อเสนอแนะฟีเจอร์ใหม่ ๆ และโหวตให้กับข้อเสนอที่มีอยู่แล้วได้ ข้อเสนอแนะจะถูกเรียงลำดับตามจำนวนคะแนนโหวต

ช่องระบบทั้งสองมีขีดจำกัดขนาด 25 MB และอยู่ภายใต้การตรวจสอบของระบบตรวจสอบเนื้อหาแบบคณะลูกขุน

## 7. การโทร

### เริ่มการโทรด้วยเสียง

1. เปิดแชทกับผู้ติดต่อที่คุณต้องการโทรหา
2. แตะไอคอนโทรศัพท์ในแถบด้านบน
3. รอจนกว่าคู่สนทนาของคุณจะรับสาย

ระหว่างการสนทนา คุณจะเห็นเส้นเวลาที่แสดงระยะเวลาการสนทนา และสามารถเข้าถึงปุ่มปิดเสียงและลำโพงได้

หากต้องการวางสาย ให้แตะปุ่มวางสายสีแดง

### เริ่มการโทรวิดีโอ

1. เปิดแชทกับผู้ติดต่อ
2. แตะไอคอนกล้องในแถบด้านบน
3. ภาพวิดีโอของคุณจะปรากฏในหน้าต่างเล็ก ส่วนภาพของคู่สนทนาจะแสดงในพื้นที่ใหญ่

คุณสามารถสลับระหว่างกล้องหน้าและกล้องหลังได้ระหว่างการสนทนา

### สายเรียกเข้า

เมื่อมีใครโทรหาคุณ หน้าต่างแจ้งเตือนพร้อมชื่อผู้โทรจะปรากฏขึ้น คุณสามารถ:

- **รับสาย** -- การสนทนาจะเริ่มต้นขึ้น
- **ปฏิเสธสาย** -- ผู้โทรจะได้รับการแจ้งเตือน

หากคุณกำลังสนทนาอยู่แล้ว สายที่โทรเข้ามาใหม่จะถูกปฏิเสธโดยอัตโนมัติ

### การโทรแบบกลุ่ม

คุณยังสามารถโทรแบบกลุ่มซึ่งมีผู้เข้าร่วมหลายคนพร้อมกันได้ การโทรจะถูกจัดการผ่านโครงสร้างการส่งต่อแบบอัจฉริยะ เพื่อไม่ให้ผู้เข้าร่วมทุกคนต้องเชื่อมต่อโดยตรงกับทุกคนอื่น การสนทนาทั้งหมดถูกเข้ารหัสตลอดเส้นทาง

### การเข้ารหัสในการโทร

การโทรทุกครั้งจะถูกเข้ารหัสด้วยกุญแจที่ใช้เพียงครั้งเดียว ซึ่งมีอยู่เฉพาะในช่วงเวลาที่สนทนาเท่านั้น หลังจากวางสาย กุญแจเหล่านี้จะถูกลบทิ้งทันที ไม่มีใครสามารถถอดรหัสบทสนทนาที่ผ่านไปแล้วในภายหลังได้

## 8. ปฏิทิน

Cleona มีปฏิทินในตัวที่ทำงานแบบเข้ารหัสและกระจายศูนย์อย่างสมบูรณ์ -- โดยไม่ต้องพึ่งพาบริการคลาวด์ใด ๆ

### มุมมอง

ปฏิทินมีมุมมองทั้งหมดห้าแบบ: วัน สัปดาห์ เดือน ปี และมุมมองงาน สลับไปมาระหว่างมุมมองเหล่านี้ได้ผ่านแท็บที่ขอบด้านบนของหน้าจอปฏิทิน

### สร้างนัดหมาย

แตะที่ช่วงเวลาใดเวลาหนึ่ง หรือใช้ปุ่มเพิ่ม เพื่อสร้างนัดหมายใหม่ คุณสามารถกรอกชื่อเรื่อง วันที่ เวลา สถานที่ และบันทึกย่อได้ นัดหมายจะถูกจัดเก็บแบบเข้ารหัสบนอุปกรณ์ของคุณ

### นัดหมายที่เกิดขึ้นซ้ำ

นัดหมายสามารถเกิดขึ้นซ้ำได้ทุกวัน ทุกสัปดาห์ ทุกเดือน หรือทุกปี คุณสามารถปรับรูปแบบได้ (เช่น ทุกวันอังคารที่สองของเดือน หรือวันที่หนึ่งของทุกเดือน) และกำหนดวันสิ้นสุดหรือจำนวนครั้งที่จะเกิดซ้ำได้

### เชิญผู้ติดต่อ

เมื่อสร้างหรือแก้ไขนัดหมาย คุณสามารถเชิญผู้ติดต่อ Cleona ของคุณได้ พวกเขาจะได้รับคำเชิญปฏิทินแบบเข้ารหัส และสามารถตอบรับ ปฏิเสธ หรือตอบว่า "อาจจะ" ได้ การเปลี่ยนแปลงใด ๆ ในนัดหมายจะถูกส่งไปยังผู้ที่ได้รับเชิญทุกคนโดยอัตโนมัติ

### แสดงสถานะว่าง/ไม่ว่าง

คุณสามารถแชร์ความพร้อมของคุณกับผู้ติดต่อได้ โดยไม่ต้องเปิดเผยรายละเอียดของนัดหมาย มีระดับความเป็นส่วนตัวสามระดับ ได้แก่ รายละเอียดแบบเต็ม แสดงเฉพาะช่วงเวลาที่ไม่ว่าง หรือซ่อนทั้งหมด คุณสามารถตั้งค่าเริ่มต้นและปรับแก้เป็นรายผู้ติดต่อได้

### การแจ้งเตือน

นัดหมายสามารถตั้งการแจ้งเตือนที่จะส่งการแจ้งเตือนระบบก่อนถึงเวลานัดหมายได้ คุณสามารถเลื่อนการแจ้งเตือนได้ตามต้องการ

### การซิงค์ปฏิทินภายนอก

Cleona สามารถซิงโครไนซ์กับบริการปฏิทินภายนอกได้:

- **CalDAV** -- เชื่อมต่อกับเซิร์ฟเวอร์ที่รองรับ CalDAV ทุกชนิด (Nextcloud, Radicale เป็นต้น)
- **Google ปฏิทิน** -- ซิงโครไนซ์ผ่าน Google Calendar API ด้วยการยืนยันตัวตนแบบ OAuth2 ที่ปลอดภัย
- **เซิร์ฟเวอร์ CalDAV ภายในเครื่อง** -- Cleona สามารถเปิดเซิร์ฟเวอร์ CalDAV บนอุปกรณ์ของคุณเองได้ ทำให้แอปปฏิทินบนเดสก์ท็อป (Thunderbird, Outlook, Apple Kalender, Evolution) สามารถซิงค์กับปฏิทิน Cleona ของคุณได้
- **ปฏิทินระบบของ Android** -- นัดหมายจาก Cleona สามารถถูกส่งต่อไปยังแอปปฏิทินที่ติดตั้งมาพร้อมเครื่องของอุปกรณ์ Android ของคุณได้
- **ไฟล์ ICS** -- นำเข้าและส่งออกนัดหมายในรูปแบบมาตรฐาน iCalendar

### ส่งออกเป็น PDF

คุณสามารถพิมพ์หรือส่งออกมุมมองปฏิทินใด ๆ ก็ตาม (วัน สัปดาห์ เดือน ปี) เป็นเอกสาร PDF ได้

## 9. แบบสำรวจ

คุณสามารถสร้างแบบสำรวจในแชทหรือกลุ่มใดก็ได้ เพื่อรับฟังความคิดเห็นหรือวางแผนนัดหมาย

### ประเภทของแบบสำรวจ

Cleona รองรับแบบสำรวจห้าประเภท:

- **เลือกคำตอบเดียว** -- ผู้เข้าร่วมเลือกได้เพียงตัวเลือกเดียว
- **เลือกได้หลายคำตอบ** -- ผู้เข้าร่วมสามารถเลือกได้หลายตัวเลือก
- **แบบสำรวจนัดหมาย** -- ค้นหาวันเวลาที่เหมาะกับทุกคน ผู้เข้าร่วมแต่ละคนทำเครื่องหมายว่าช่วงเวลาใดว่าง อาจจะว่าง หรือไม่ว่าง
- **มาตราส่วน** -- ให้คะแนนสิ่งใดสิ่งหนึ่งบนมาตราส่วนตัวเลข (เช่น 1 ถึง 5)
- **ข้อความอิสระ** -- ผู้เข้าร่วมเขียนคำตอบของตนเองได้อย่างอิสระ

### สร้างแบบสำรวจ

เปิดแชทและแตะไอคอนแบบสำรวจ (หรือใช้เมนูแนบไฟล์) เลือกประเภทของแบบสำรวจ ตั้งคำถามและตัวเลือกคำตอบ แล้วส่งแบบสำรวจออกไป แบบสำรวจจะปรากฏเป็นข้อความในแชท

### การโหวต

แตะที่แบบสำรวจเพื่อโหวต คุณสามารถเปลี่ยนหรือยกเลิกคะแนนโหวตของคุณได้ทุกเมื่อ

### การโหวตแบบไม่ระบุตัวตน

แบบสำรวจสามารถตั้งค่าให้รองรับการโหวตแบบไม่ระบุตัวตนได้ เมื่อเปิดใช้งาน คะแนนโหวตจะไม่ระบุตัวตนในเชิงเข้ารหัส -- ไม่มีใครแม้แต่ผู้สร้างแบบสำรวจเองก็ไม่สามารถรู้ได้ว่าใครโหวตให้อะไร แต่จำนวนคะแนนโหวตยังคงมองเห็นได้ตามปกติ

### จากแบบสำรวจนัดหมายสู่ปฏิทิน

เมื่อแบบสำรวจนัดหมายเสร็จสิ้นแล้ว นัดหมายที่ได้รับคะแนนสูงสุดสามารถถูกแปลงเป็นรายการในปฏิทินได้ทันทีเพียงแตะปุ่มเดียว

## 10. หลายตัวตน

### เหตุใดจึงต้องมีหลายตัวตน?

ลองนึกภาพว่าคุณต้องการแยกชีวิตการทำงานออกจากชีวิตส่วนตัว -- คล้ายกับการมีเบอร์โทรศัพท์สองเบอร์ แต่ไม่ต้องมีโทรศัพท์สองเครื่อง ใน Cleona คุณสามารถใช้หลายตัวตนบนอุปกรณ์เครื่องเดียวได้ แต่ละตัวตนมีชื่อ รูปโปรไฟล์ ผู้ติดต่อ และบทสนทนาเป็นของตนเอง

### สร้างตัวตนใหม่

1. ในแถบด้านบน คุณจะเห็นตัวตนปัจจุบันของคุณแสดงเป็นแท็บ
2. แตะเครื่องหมายบวก (+) ทางด้านขวาของแท็บตัวตนของคุณ
3. ตั้งชื่อสำหรับตัวตนใหม่
4. เสร็จเรียบร้อย -- ตัวตนใหม่จะเริ่มใช้งานได้ทันที

### สลับระหว่างตัวตน

เพียงแตะที่แท็บตัวตนในแถบด้านบน การสลับจะเกิดขึ้นทันที -- ไม่มีการรอ ไม่มีการโหลดใหม่

### ทั้งหมดทำงานพร้อมกัน

ประเด็นสำคัญที่ควรทราบ: ตัวตนทั้งหมดของคุณทำงานอยู่พร้อมกันตลอดเวลา แม้ในขณะที่คุณกำลังแสดงเป็นตัวตน "การทำงาน" ตัวตน "ส่วนตัว" ของคุณก็ยังคงรับข้อความอยู่ตามปกติ คุณจะไม่พลาดข้อความใด ๆ ไม่ว่าคุณจะเลือกใช้ตัวตนใดอยู่ก็ตาม

### หน้ารายละเอียดตัวตน

เมื่อคุณแตะที่แท็บของตัวตนที่กำลังใช้งานอยู่ หน้ารายละเอียดจะเปิดขึ้น ที่นี่คุณสามารถ:

- แสดง QR-Code ของคุณสำหรับผู้ติดต่อ
- เปลี่ยนหรือลบรูปโปรไฟล์ของคุณ
- เพิ่มคำอธิบายโปรไฟล์
- เปลี่ยนชื่อที่แสดงของคุณ
- เลือกธีมสำหรับตัวตนนี้
- ลบตัวตนนี้ หากคุณไม่ต้องการใช้งานอีกต่อไป

### ลบตัวตน

เมื่อคุณลบตัวตนหนึ่ง ผู้ติดต่อของคุณจะได้รับการแจ้งเตือนเกี่ยวกับเรื่องนี้ ตัวตนและข้อมูลทั้งหมดที่เกี่ยวข้องจะถูกลบออกจากอุปกรณ์ของคุณ กระบวนการนี้ไม่สามารถย้อนกลับได้

## 11. หลายอุปกรณ์

### ใช้งาน Cleona บนหลายอุปกรณ์

คุณสามารถใช้ตัวตนเดียวกันบนอุปกรณ์ได้สูงสุด 5 เครื่องพร้อมกัน อุปกรณ์หนึ่งเครื่องจะเป็นอุปกรณ์หลัก (เก็บ Seed-Phrase ไว้) ส่วนอุปกรณ์อื่น ๆ จะถูกเชื่อมโยงเข้ากับอุปกรณ์หลักนี้

### เชื่อมโยงอุปกรณ์ใหม่

1. เปิดการตั้งค่าบนอุปกรณ์หลักของคุณ
2. ไปที่ "อุปกรณ์ที่เชื่อมโยง"
3. เลือก "เชื่อมโยงอุปกรณ์ใหม่"
4. ติดตั้ง Cleona บนอุปกรณ์เครื่องใหม่ และเลือก "เชื่อมโยงกับอุปกรณ์ที่มีอยู่แล้ว" เมื่อเริ่มใช้งานครั้งแรก
5. สแกน QR-Code สำหรับการจับคู่ที่แสดงบนอุปกรณ์หลักของคุณ หรือใช้ลิงก์จับคู่แทน

อุปกรณ์ที่ถูกเชื่อมโยงจะได้รับใบรับรองมอบอำนาจจากอุปกรณ์หลัก ข้อความที่ถูกส่งจากอุปกรณ์ที่เชื่อมโยงจะถูกลงลายเซ็นด้วยกุญแจที่ได้รับมอบอำนาจในเชิงเข้ารหัส เพื่อให้ผู้ติดต่อสามารถตรวจสอบได้ว่าข้อความนั้นมาจากตัวตนของคุณจริง

### หลักการทำงาน

- อุปกรณ์หลักเก็บ Seed-Phrase และกุญแจหลักของคุณไว้
- อุปกรณ์ที่เชื่อมโยงจะได้รับเพียงกุญแจลายเซ็นที่ได้จากการคำนวณและใบรับรองมอบอำนาจเท่านั้น -- อุปกรณ์เหล่านี้จะไม่มีทางได้รับ Seed-Phrase ตัวจริงเลย
- อุปกรณ์ทั้งหมดใช้ตัวตนและผู้ติดต่อร่วมกัน ข้อความจะมาถึงทุกอุปกรณ์พร้อมกัน
- ใบรับรองมอบอำนาจจะถูกต่ออายุโดยอัตโนมัติก่อนหมดอายุ

### การจัดการอุปกรณ์

เปิดการตั้งค่าและไปที่ "อุปกรณ์ที่เชื่อมโยง" เพื่อดูอุปกรณ์ที่เชื่อมโยงทั้งหมดของคุณ สถานะ และกิจกรรมล่าสุดของแต่ละเครื่อง คุณสามารถเพิกถอนอุปกรณ์ที่เชื่อมโยงได้ทุกเมื่อ หากอุปกรณ์นั้นสูญหายหรือถูกขโมย

### การหมุนเวียนกุญแจฉุกเฉิน

หากคุณสงสัยว่าอุปกรณ์เครื่องใดเครื่องหนึ่งถูกบุกรุก คุณสามารถเริ่มกระบวนการหมุนเวียนกุญแจฉุกเฉินได้ กระบวนการนี้จะสร้างกุญแจใหม่ทั้งหมด และการหมุนเวียนกุญแจนี้ต้องได้รับการยืนยันจากเสียงข้างมากของอุปกรณ์อื่น ๆ ของคุณ วิธีนี้ป้องกันไม่ให้อุปกรณ์ที่ถูกขโมยเพียงเครื่องเดียวสามารถหมุนเวียนกุญแจได้ตามอำเภอใจ

## 12. การกู้คืนข้อมูล

### ใช้ Seed-Phrase

หากคุณทำอุปกรณ์หายหรือกำลังตั้งค่าเครื่องใหม่:

1. ติดตั้ง Cleona บนอุปกรณ์เครื่องใหม่
2. เลือก "กู้คืน" เมื่อเริ่มใช้งานครั้งแรก
3. ป้อนคำ 24 คำของคุณ
4. Cleona จะกู้คืนตัวตนของคุณและติดต่อผู้ติดต่อเดิมของคุณโดยอัตโนมัติ
5. ผู้ติดต่อของคุณจะตอบกลับด้วยข้อมูลผู้ติดต่อ การเป็นสมาชิกกลุ่ม และประวัติการสนทนาของคุณ

การกู้คืนจะเกิดขึ้นเป็นสามขั้นตอน:
- อันดับแรก ผู้ติดต่อและกลุ่มของคุณจะกลับคืนมา
- จากนั้นข้อความ 50 รายการล่าสุดจากทุกบทสนทนา
- สุดท้ายคือประวัติข้อความแบบสมบูรณ์ทั้งหมด

เพียงแค่มีผู้ติดต่อของคุณคนใดคนหนึ่งออนไลน์อยู่ก็เพียงพอแล้วที่จะทำให้การกู้คืนสำเร็จ

### การกู้คืนผ่าน Guardian (บุคคลที่ไว้วางใจ)

คุณสามารถกำหนดบุคคลที่ไว้วางใจได้สูงสุดห้าคนให้เป็น "Guardian" ได้ โดยกุญแจกู้คืนของคุณจะถูกแบ่งออกเป็นห้าส่วน แต่ละ Guardian จะได้รับหนึ่งส่วน ในการกู้คืนตัวตนของคุณ จำเป็นต้องใช้เพียงสามในห้าส่วนเท่านั้น

นั่นหมายความว่า แม้ว่าคุณจะทำ Seed-Phrase หาย Guardian สามคนจากทั้งหมดก็สามารถร่วมกันกู้คืนบัญชีของคุณได้ Guardian เพียงคนเดียวไม่สามารถเข้าถึงข้อมูลของคุณได้เลย -- ต้องใช้อย่างน้อยสามคนเสมอ

วิธีตั้งค่า Guardian:
1. เปิดการตั้งค่า
2. ไปที่ "ความปลอดภัย"
3. เลือก "Guardian Recovery"
4. เลือกผู้ติดต่อที่ไว้วางใจห้าคน

### เหตุใดผู้ติดต่อของคุณจึงเป็นระบบสำรองข้อมูลของคุณ

ในแอปส่งข้อความทั่วไป ข้อมูลของคุณจะถูกเก็บไว้บนเซิร์ฟเวอร์ของผู้ให้บริการ แต่ใน Cleona ไม่มีเซิร์ฟเวอร์เลย -- ผู้ติดต่อของคุณจะรับหน้าที่นี้แทน เมื่อคุณส่งข้อความ ผู้ติดต่อร่วมกันจะเก็บสำเนาที่เข้ารหัสไว้ในกรณีที่ผู้รับกำลังออฟไลน์อยู่ในขณะนั้น เมื่อมีการกู้คืนข้อมูล ผู้ติดต่อของคุณจะส่งข้อมูลของคุณกลับคืนให้

นั่นหมายความว่า ยิ่งคุณมีผู้ติดต่อที่ใช้งานอยู่จริงมากเท่าไร ระบบสำรองข้อมูลของคุณก็จะยิ่งน่าเชื่อถือมากขึ้นเท่านั้น เพียงมีผู้ติดต่อหนึ่งคนที่ออนไลน์อยู่เป็นประจำก็เพียงพอแล้วสำหรับการกู้คืนที่สำเร็จ

## 13. การตั้งค่า

คุณสามารถเข้าถึงการตั้งค่าได้ผ่านไอคอนรูปเฟืองที่มุมขวาบน

### การแจ้งเตือนและเสียงเรียกเข้า

- เลือกจากเสียงเรียกเข้าหกแบบสำหรับสายที่โทรเข้ามา
- ตั้งค่าเสียงแจ้งเตือนข้อความ
- บนอุปกรณ์ Android คุณยังสามารถเปิดหรือปิดการสั่นเพิ่มเติมได้

### ธีม (Skins)

Cleona มีธีมให้เลือกสิบแบบ ได้แก่ Teal, Ocean, Sunset, Forest, Amethyst, Fire, Storm, Slate, Gold และ Contrast ธีม Contrast ตรงตามมาตรฐานการเข้าถึงระดับสูงสุด (WCAG AAA) และอ่านง่ายเป็นพิเศษสำหรับผู้ที่มีความบกพร่องทางสายตา

แต่ละตัวตนสามารถมีธีมของตนเองได้ คุณสามารถเปลี่ยนธีมได้ในหน้ารายละเอียดตัวตน (แตะที่แท็บตัวตนที่กำลังใช้งานอยู่)

นอกจากนี้ในการตั้งค่า หัวข้อ "รูปลักษณ์" คุณยังสามารถสลับระหว่างธีมสว่าง ธีมมืด และธีมตามระบบได้อีกด้วย

### เปลี่ยนภาษา

Cleona มีให้บริการใน 33 ภาษา รวมถึงภาษาที่เขียนจากขวาไปซ้าย (เช่น อาหรับ ฮีบรู) เปลี่ยนภาษาได้ในการตั้งค่า หัวข้อ "ภาษา"

### ขีดจำกัดพื้นที่จัดเก็บ

คุณสามารถกำหนดได้ว่า Cleona จะใช้พื้นที่จัดเก็บบนอุปกรณ์ของคุณได้มากเท่าใด (ระหว่าง 100 MB ถึง 2 GB) เมื่อถึงขีดจำกัดแล้ว สื่อเก่า ๆ จะถูกย้ายออกไปเก็บที่อื่นหรือถูกลบโดยอัตโนมัติ -- ข้อความตัวอักษรจะยังคงอยู่เสมอ

### การจัดเก็บสื่อถาวร

หากที่บ้านคุณมีที่เก็บข้อมูลบนเครือข่าย (NAS) หรือโฟลเดอร์แชร์ Cleona สามารถย้ายสื่อของคุณไปเก็บไว้ที่นั่นโดยอัตโนมัติได้ รองรับ SMB, SFTP, FTPS และ WebDAV

ระบบการจัดเก็บแบบขั้นบันไดทำงานดังนี้:
- 30 วันแรก: ทุกอย่างยังอยู่บนอุปกรณ์ของคุณ
- หลังจาก 30 วัน: ภาพตัวอย่างจะยังคงอยู่บนอุปกรณ์ ในขณะที่ไฟล์ต้นฉบับจะถูกจัดเก็บถาวร
- หลังจาก 90 วัน: จะเหลือเพียงภาพตัวอย่างขนาดเล็กบนอุปกรณ์เท่านั้น
- หลังจาก 1 ปี: จะเหลือเพียงตัวยึดตำแหน่งเท่านั้น ไฟล์ต้นฉบับจะถูกเก็บไว้อย่างปลอดภัยในที่จัดเก็บถาวร

คุณสามารถแตะที่สื่อที่ถูกจัดเก็บถาวรได้ทุกเมื่อเพื่อดึงกลับมา -- โดยมีเงื่อนไขว่าคุณต้องเชื่อมต่อกับเครือข่ายภายในบ้านของคุณอยู่ สื่อที่สำคัญเป็นพิเศษสามารถปักหมุดไว้เพื่อไม่ให้ถูกย้ายออกไปเก็บที่อื่นได้

### การถอดเสียงข้อความเสียง

เมื่อเปิดใช้งาน ข้อความเสียงของคุณจะถูกแปลงเป็นข้อความตัวอักษรบนอุปกรณ์ของคุณเองในเครื่อง (ด้วยโมเดลโอเพนซอร์ส Whisper) ข้อความที่ถอดเสียงแล้วจะถูกส่งไปพร้อมกับไฟล์เสียงถึงคู่สนทนาของคุณ กระบวนการถอดเสียงทั้งหมดเกิดขึ้นบนอุปกรณ์ของคุณเอง -- ไม่มีข้อมูลใดถูกส่งไปยังบริการภายนอกเลย

### ดาวน์โหลดอัตโนมัติ

คุณสามารถตั้งค่าได้ว่าตั้งแต่ขนาดเท่าใดที่สื่อจะถูกดาวน์โหลดโดยอัตโนมัติ ตัวอย่างเช่น คุณสามารถให้ระบบดาวน์โหลดรูปภาพอัตโนมัติ แต่เลือกดาวน์โหลดวิดีโอขนาดใหญ่ด้วยตนเอง

### อุปกรณ์ที่เชื่อมโยง

จัดการอุปกรณ์ที่เชื่อมโยงของคุณได้ในส่วนนี้ของการตั้งค่า ดูรายละเอียดเพิ่มเติมได้ที่บทหลายอุปกรณ์

## 14. ความปลอดภัย

### การเข้ารหัสแบบต้านทานควอนตัม (Post-Quantum) คืออะไร?

การเข้ารหัสในปัจจุบันอาศัยปัญหาทางคณิตศาสตร์ที่ยากมากสำหรับคอมพิวเตอร์ทั่วไปในการแก้ไข คอมพิวเตอร์ควอนตัมในอนาคตอาจสามารถแก้ปัญหาบางส่วนเหล่านี้ได้อย่างรวดเร็ว การเข้ารหัสแบบต้านทานควอนตัมใช้กระบวนการเสริมที่สามารถทนทานต่อคอมพิวเตอร์ควอนตัมได้เช่นกัน

Cleona ผสมผสานทั้งสองแนวทางเข้าด้วยกัน: การเข้ารหัสแบบดั้งเดิมเพื่อความน่าเชื่อถือ และกระบวนการต้านทานควอนตัมเพื่อความมั่นคงในอนาคต ด้วยวิธีนี้คุณจะได้รับการปกป้องจากภัยคุกคามทั้งในปัจจุบันและอนาคตไปพร้อมกัน

ข้อความแต่ละฉบับจะถูกสร้างกุญแจของตัวเองขึ้นมาโดยเฉพาะ แม้ว่าผู้โจมตีจะสามารถถอดรหัสกุญแจของข้อความหนึ่งได้ ก็จะไม่สามารถใช้กุญแจนั้นอ่านข้อความอื่นใดได้เลย

### เหตุใดการไม่มีเซิร์ฟเวอร์จึงปลอดภัยกว่า

ในแอปส่งข้อความทั่วไป ข้อความของคุณจะวิ่งผ่านเซิร์ฟเวอร์ของผู้ให้บริการ แม้ว่าข้อความเหล่านั้นอาจถูกเข้ารหัสไว้ แต่ผู้ให้บริการยังคงสามารถเข้าถึงข้อมูลเมทาดาต้าได้ (ใครสื่อสารกับใคร เมื่อไร บ่อยแค่ไหน จากที่ใด) และในบางกรณีอาจต้องส่งมอบข้อมูลเหล่านี้ตามคำสั่งศาล

ใน Cleona ไม่มีจุดศูนย์กลางเช่นนั้นเลย ข้อความของคุณเดินทางตรงจากอุปกรณ์หนึ่งไปยังอีกอุปกรณ์หนึ่ง ไม่มีที่ใดที่ข้อมูลเมทาดาต้าทั้งหมดจะมารวมกัน ไม่มีใครสามารถประกอบพฤติกรรมการสื่อสารของคุณขึ้นมาใหม่ได้จากจุดข้อมูลเพียงจุดเดียว

### จะเกิดอะไรขึ้นเมื่อคุณออฟไลน์?

เมื่อคุณส่งข้อความและผู้รับกำลังออฟไลน์อยู่:

1. Cleona จะพยายามส่งข้อความโดยตรงก่อนเป็นอันดับแรก
2. หากไม่สำเร็จ ข้อความจะถูกส่งต่อผ่านผู้ติดต่อร่วมกัน
3. ในเวลาเดียวกัน ข้อความจะถูกแบ่งเป็นชิ้นส่วนที่เข้ารหัสแล้วและกระจายไปยังหลายโหนดในเครือข่าย (คล้ายกับจิ๊กซอว์ที่ประกอบด้วย 10 ชิ้น ซึ่งเพียง 7 ชิ้นก็เพียงพอที่จะประกอบภาพให้สมบูรณ์)
4. ข้อความจะถูกเก็บรักษาไว้นานถึง 7 วัน

ทันทีที่ผู้รับกลับมาออนไลน์อีกครั้ง ข้อความจะถูกส่งถึงทันที คุณจะได้รับการยืนยันเมื่อข้อความของคุณถูกส่งถึงแล้ว

### การต้านทานการเซ็นเซอร์

หากเครือข่ายของคุณปิดกั้นวิธีการเชื่อมต่อมาตรฐาน (UDP) Cleona จะสลับไปใช้การส่งข้อมูลทางเลือกโดยอัตโนมัติ (TLS) ซึ่งตรวจจับและปิดกั้นได้ยากกว่า กระบวนการนี้เกิดขึ้นโดยอัตโนมัติอย่างโปร่งใส -- คุณไม่จำเป็นต้องตั้งค่าใด ๆ เลย

### การจัดเก็บกุญแจอย่างปลอดภัย

บนแพลตฟอร์มที่รองรับ Cleona จะจัดเก็บกุญแจเข้ารหัสของคุณไว้ในพวงกุญแจที่ปลอดภัยของระบบปฏิบัติการ (Android Keystore, iOS Keychain, macOS Keychain) ในที่ที่รองรับ วิธีนี้จะให้การปกป้องกุญแจของคุณในระดับฮาร์ดแวร์

### การเข้ารหัสฐานข้อมูล

ข้อความ ผู้ติดต่อ และการตั้งค่าทั้งหมดของคุณถูกจัดเก็บแบบเข้ารหัสบนอุปกรณ์ของคุณ แม้ว่าจะมีใครสามารถเข้าถึงระบบไฟล์ของคุณได้ ก็จะไม่สามารถอ่านสิ่งใดได้เลยหากไม่มีกุญแจเข้ารหัสของคุณ กุญแจนี้ถูกคำนวณมาจากตัวตนของคุณและมีอยู่เฉพาะบนอุปกรณ์ของคุณเท่านั้น

### เครือข่ายปิด

Cleona ทำงานในรูปแบบเครือข่ายปิด แพ็กเก็ตเครือข่ายทุกชิ้นถูกยืนยันความถูกต้อง เพื่อให้มีเพียงอุปกรณ์ Cleona ที่ถูกต้องตามกฎหมายเท่านั้นที่สามารถเข้าร่วมเครือข่ายได้ วิธีนี้ป้องกันไม่ให้บุคคลภายนอกแทรกข้อความปลอมหรือดักฟังการสื่อสารในเครือข่ายได้

## 15. การอัปเดตซอฟต์แวร์

### ฉันจะได้รับการอัปเดตได้อย่างไร?

Cleona สามารถอัปเดตได้หลายวิธี เป้าหมายคือให้คุณยังคงได้รับการอัปเดตแม้ว่าช่องทางการเผยแพร่บางช่องทางจะขัดข้องหรือถูกปิดกั้นก็ตาม:

1. **App Store / Play Store:** หากคุณติดตั้ง Cleona ผ่าน App Store คุณจะได้รับการอัปเดตตามปกติผ่านทางร้านค้านั้น
2. **GitHub Releases:** บนหน้า GitHub ของโปรเจกต์ คุณจะพบแพ็กเกจติดตั้งที่ลงลายเซ็นรับรองแล้วสำหรับทุกแพลตฟอร์ม
3. **การอัปเดตภายในเครือข่าย (In-Network-Updates):** หากผู้ใช้ Cleona คนอื่นในเครือข่ายของคุณมีเวอร์ชันล่าสุดอยู่แล้ว Cleona สามารถรับการอัปเดตโดยตรงผ่านเครือข่าย P2P ได้ -- โดยไม่ต้องพึ่งเซิร์ฟเวอร์ภายนอกเลย ในกระบวนการนี้ เวอร์ชันใหม่จะถูกแบ่งออกเป็นชิ้นส่วนที่มีการแก้ไขข้อผิดพลาดและกระจายไปยังหลายโหนด อุปกรณ์ของคุณจะรวบรวมชิ้นส่วนให้เพียงพอและประกอบการอัปเดตขึ้นมาใหม่ ความถูกต้องแท้จริงจะถูกตรวจสอบด้วยลายเซ็น Ed25519 ของนักพัฒนา
4. **ลิงก์คำเชิญ:** คุณสามารถสร้างลิงก์คำเชิญที่มีทุกสิ่งที่ผู้ใช้ใหม่ต้องการเพื่อติดตั้ง Cleona และเชื่อมต่อกับเครือข่ายได้
5. **การถ่ายโอนทางกายภาพ:** ในสภาพแวดล้อมที่ไม่มีอินเทอร์เน็ต คุณสามารถส่งต่อ Cleona ผ่าน USB หรือเครือข่ายภายในให้ผู้อื่นได้

### การแจ้งเตือนอัปเดต

เมื่อมีอัปเดตใหม่พร้อมใช้งาน Cleona จะแสดงการแจ้งเตือนบนหน้าจอเริ่มต้นให้คุณเห็น หากอัปเดตนั้นสามารถรับผ่านเครือข่ายได้ด้วย (In-Network-Update) คุณจะมีตัวเลือกให้ดาวน์โหลดโดยตรงจากเครือข่ายได้

### การกระจายไบนารี

ตามค่าเริ่มต้น อุปกรณ์ของคุณจะช่วยกระจายการอัปเดตไปยังผู้ใช้อื่นในเครือข่าย หากคุณไม่ต้องการเช่นนั้น คุณสามารถปิดใช้งานฟังก์ชันนี้ได้ในการตั้งค่า หัวข้อ "เครือข่าย" การใช้พื้นที่จัดเก็บสำหรับชิ้นส่วนอัปเดตมีขีดจำกัด (5 MB บนอุปกรณ์มือถือ, 20 MB บนอุปกรณ์เดสก์ท็อป) และจะถูกล้างเป็นระยะอย่างสม่ำเสมอ

### การตรวจสอบลายเซ็น

การอัปเดตทุกครั้งจะถูกลงลายเซ็นในเชิงเข้ารหัส Cleona จะตรวจสอบลายเซ็นโดยอัตโนมัติก่อนที่จะติดตั้งการอัปเดตใด ๆ วิธีนี้ทำให้มั่นใจได้ว่าจะมีเพียงอัปเดตจากนักพัฒนาอย่างเป็นทางการเท่านั้นที่จะได้รับการยอมรับ -- แม้ว่าอัปเดตนั้นจะได้รับผ่านเครือข่าย P2P ก็ตาม

## 16. คำถามที่พบบ่อย

### "ฉันสามารถใช้ Cleona โดยไม่มีอินเทอร์เน็ตได้หรือไม่?"

ไม่ได้ Cleona จำเป็นต้องมีการเชื่อมต่อเครือข่ายเพื่อส่งและรับข้อความ อย่างไรก็ตาม คุณไม่จำเป็นต้องออนไลน์พร้อมกับคู่สนทนาของคุณในเวลาเดียวกัน: ข้อความที่ถูกส่งในขณะที่ผู้รับออฟไลน์อยู่จะถูกเก็บไว้ชั่วคราวและส่งถึงโดยอัตโนมัติทันทีที่ทั้งสองฝ่ายเชื่อมต่อกันอีกครั้ง ในเครือข่ายภายใน (เช่น WLAN เดียวกัน) คุณยังสามารถสื่อสารกันได้โดยไม่ต้องเชื่อมต่ออินเทอร์เน็ตเลยด้วยซ้ำ

### "จะเกิดอะไรขึ้นถ้าฉันทำ Seed-Phrase หาย?"

หากคุณได้ตั้งค่า Guardian ไว้แล้ว บุคคลที่ไว้วางใจสามในห้าคนสามารถร่วมกันกู้คืนการเข้าถึงของคุณได้ หากไม่มี Guardian และไม่มี Seed-Phrase ก็น่าเสียดายที่ไม่มีทางใดจะกู้คืนตัวตนของคุณกลับมาได้เลย ด้วยเหตุนี้ การเก็บรักษาคำ 24 คำไว้อย่างปลอดภัยจึงมีความสำคัญอย่างยิ่ง

### "มีใครสามารถอ่านข้อความของฉันได้หรือไม่?"

ไม่ได้ ข้อความแต่ละฉบับจะถูกเข้ารหัสด้วยกุญแจที่ใช้เพียงครั้งเดียวและใช้ได้เฉพาะกับข้อความนั้นเท่านั้น มีเพียงคุณและคู่สนทนาของคุณเท่านั้นที่สามารถถอดรหัสข้อความได้ ไม่มีเซิร์ฟเวอร์กลาง ไม่มีกุญแจแม่บทที่ใช้ได้กับทุกข้อความ และไม่มีช่องทางเข้าถึงสำหรับนักพัฒนาเลย แม้ว่าอุปกรณ์ใดจะส่งต่อข้อความระหว่างทาง อุปกรณ์นั้นก็จะเห็นเพียงข้อมูลที่เข้ารหัสไว้เท่านั้น

### "เหตุใดฉันจึงไม่ต้องใช้เบอร์โทรศัพท์?"

เพราะตัวตนของคุณเป็นสิ่งที่กำหนดขึ้นจากการเข้ารหัสล้วน ๆ แทนที่จะใช้เบอร์โทรศัพท์หรือที่อยู่อีเมลที่เชื่อมโยงกับชื่อจริงของคุณ คู่กุญแจที่ถูกสร้างขึ้นบนอุปกรณ์ของคุณเองจะเป็นตัวระบุตัวตนของคุณแทน คุณเพิ่มผู้ติดต่อผ่าน QR-Code, NFC หรือลิงก์ -- ไม่ใช่ผ่านสมุดโทรศัพท์ นั่นหมายถึงความเป็นส่วนตัวที่มากขึ้น เพราะบัญชี Messenger ของคุณไม่ได้ผูกติดอยู่กับตัวตนจริงของคุณ

### "ฉันจะหาคนอื่นบน Cleona ได้อย่างไร?"

Cleona จงใจไม่มีฟังก์ชันค้นหาผู้ติดต่อด้วยเบอร์โทรศัพท์หรือชื่อ -- เพราะนั่นจะเป็นปัญหาด้านความเป็นส่วนตัว แทนที่จะเป็นเช่นนั้น คุณแลกเปลี่ยนข้อมูลผู้ติดต่อกันโดยตรง: ผ่าน QR-Code, NFC, ลิงก์ cleona:// หรือในช่องสาธารณะ วิธีนี้เปรียบเสมือนการแลกนามบัตรกัน แทนที่จะค้นหาในสมุดโทรศัพท์

### "Cleona ใช้งานได้ในต่างประเทศหรือไม่?"

ได้ ตราบใดที่คุณมีการเชื่อมต่ออินเทอร์เน็ต Cleona จะทำงานได้ทุกที่ทั่วโลก เนื่องจากไม่มีเซิร์ฟเวอร์กลาง บริการนี้จึงไม่สามารถถูกปิดกั้นสำหรับประเทศใดประเทศหนึ่งได้เช่นกัน นอกจากนี้ Cleona ยังมีกลไกสำรองต้านทานการเซ็นเซอร์: หากการเชื่อมต่อปกติ (UDP) ถูกปิดกั้น Cleona จะสลับไปใช้การส่งข้อมูลทางเลือกโดยอัตโนมัติ (TLS) ซึ่งตรวจจับและปิดกั้นได้ยากกว่า

### "Cleona ฟรีหรือไม่?"

ใช่ Cleona ใช้งานได้ฟรีและไม่มีโฆษณา เนื่องจากไม่มีเซิร์ฟเวอร์กลาง จึงไม่มีค่าใช้จ่ายด้านเซิร์ฟเวอร์ในการดำเนินงานเช่นกัน ในแอปคุณจะพบหัวข้อ "บริจาค" ซึ่งเป็นช่องทางให้คุณสนับสนุนการพัฒนาได้ตามความสมัครใจ

### "ข้อความของฉันมีสัญลักษณ์นาฬิกา -- หมายความว่าอย่างไร?"

หมายความว่าข้อความนั้นยังไม่ถูกส่งถึงผู้รับ คู่สนทนาของคุณน่าจะกำลังออฟไลน์อยู่ในขณะนี้ ทันทีที่ข้อความถูกส่งถึงแล้ว สัญลักษณ์จะเปลี่ยนไป ข้อความจะถูกเก็บรักษาไว้เพื่อรอการส่งได้นานถึง 7 วัน

### "ฉันสามารถย้ายจาก WhatsApp มาใช้ Cleona ได้หรือไม่?"

ได้ แต่คุณจะไม่สามารถโอนย้ายประวัติแชทจาก WhatsApp มาได้ Cleona และ WhatsApp เป็นระบบที่แตกต่างกันโดยพื้นฐาน คุณจะต้องเพิ่มผู้ติดต่อแต่ละคนใน Cleona ทีละคน วิธีที่ง่ายที่สุดคือโพสต์ลิงก์ cleona:// ของคุณลงในกลุ่ม WhatsApp แล้วขอให้คนอื่น ๆ เพิ่มคุณเป็นผู้ติดต่อจากที่นั่น

### "ฉันสามารถใช้ Cleona บนหลายอุปกรณ์พร้อมกันได้หรือไม่?"

ได้ คุณสามารถเชื่อมโยงอุปกรณ์ได้สูงสุด 5 เครื่องด้วยตัวตนเดียวกัน อุปกรณ์หนึ่งเครื่องจะเป็นอุปกรณ์หลัก (เก็บ Seed-Phrase ไว้) ส่วนอุปกรณ์อื่น ๆ จะถูกเชื่อมโยงผ่านกระบวนการจับคู่ที่ปลอดภัย อุปกรณ์ทั้งหมดใช้ตัวตน ผู้ติดต่อ และบทสนทนาร่วมกัน ดูรายละเอียดเพิ่มเติมได้ที่บทหลายอุปกรณ์

### "ฉันจะได้รับการอัปเดตอย่างไรหาก App Store ถูกปิดกั้น?"

Cleona สามารถรับการอัปเดตโดยตรงผ่านเครือข่าย P2P ได้ โดยไม่ต้องพึ่งพา App Store เว็บไซต์ หรือเซิร์ฟเวอร์ดาวน์โหลดใด ๆ เลย หากผู้ใช้คนอื่นในเครือข่ายมีเวอร์ชันล่าสุดอยู่แล้ว อุปกรณ์ของคุณสามารถโหลดการอัปเดตจากที่นั่นได้ ความถูกต้องแท้จริงจะถูกตรวจสอบด้วยลายเซ็นดิจิทัลของนักพัฒนา หรืออีกทางหนึ่ง ผู้ติดต่อของคุณสามารถส่งต่อแอปให้คุณผ่านลิงก์คำเชิญหรือ USB ก็ได้ อ่านรายละเอียดเพิ่มเติมได้ในบท "การอัปเดตซอฟต์แวร์"
