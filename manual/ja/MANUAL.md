## 1. Cleonaとは何ですか？

### あなたのメッセンジャー、あなたのデータ

Cleona Chatは、中央サーバーを一切使わずに動作するメッセンジャーです。
あなたのメッセージは、企業の本社を経由することも、クラウドを介することも、
データセンターを通ることもなく、あなたの端末から相手の端末へ直接届きます。
間に立つ企業がそもそも存在しないため、どの企業もあなたのメッセージを
読んだり、保存したり、第三者に渡したりすることはできません。

### アカウント不要、電話番号も不要

Cleonaでは、ログインするために電話番号もメールアドレスも必要ありません。
あなたのアイデンティティは、初回起動時に端末上で自動的に生成される暗号鍵ペアで
構成されています。つまり、あなた自身が連絡先情報を共有しない限り、
電話番号やメールアドレスからあなたを特定することは誰にもできません。

### 将来にわたって安全な暗号化

Cleonaはいわゆるポスト量子暗号を使用しています。つまり、将来の量子コンピューターを
もってしても、あなたのメッセージを解読することはできません。詳細を理解する
必要はありません -- 重要なのは、あなたのコミュニケーションが現在の技術水準に
おいて最善の形で保護されているということだけです。

### サーバーなしでどうやって動くのですか？

あなたとあなたの連絡先が一緒にネットワークを形成していると想像してください。
それぞれの端末がメッセージの中継を手伝います。相手がオンラインであれば、
メッセージは直接届きます。相手がオフラインの場合は、共通の連絡先が
メッセージを一時的に保存し、受信者が再びオンラインになった時点で配達します。
つまり、あなたの連絡先は同時にあなたのネットワークでもあるのです。

### 対応プラットフォーム

CleonaはAndroid、iOS、macOS、Linux、Windowsで利用できます。

## 2. はじめに

### アプリのインストール

**Android:**
1. Cleonaのウェブサイトまたは GitHub Releases からAPKファイルをダウンロードします。
2. スマートフォンでファイルを開きます。必要に応じて、提供元不明のアプリの
   インストールを許可してください（Androidが自動的に確認を求めます）。
3. 「インストール」をタップし、インストールが完了するまで待ちます。

**iOS:**
1. iPhoneでTestFlightの招待リンクを開きます。
2. 「インストール」をタップします。TestFlightはAppleが提供するベータ版アプリ
   配布のための公式な仕組みです。
3. インストール後、ホーム画面にCleonaが表示されます。

**macOS:**
1. Cleonaのウェブサイトまたは GitHub Releases からDMGファイルをダウンロードします。
2. DMGを開き、Cleonaを「アプリケーション」フォルダにドラッグします。
3. 初回起動時、macOSが「開発元が確認されたアプリ」を開くかどうか確認する
   場合があります -- その場合は許可してください。

**Linux (Ubuntu/Debian):**
1. Cleonaのウェブサイトまたは GitHub Releases から.debファイルをダウンロードします。
2. ダブルクリックでインストールするか、ターミナルで次を実行します: `sudo dpkg -i cleona-chat_VERSION_amd64.deb`
3. アプリケーションメニューから、またはターミナルで `cleona-chat` と入力してCleonaを起動します。

**Linux (Fedora/openSUSE):**
1. Cleonaのウェブサイトまたは GitHub Releases から.rpmファイルをダウンロードします。
2. 次のコマンドでインストールします: `sudo dnf install cleona-chat-VERSION.x86_64.rpm`
3. アプリケーションメニューから、またはターミナルで `cleona-chat` と入力してCleonaを起動します。

**Linux (全ディストリビューション対応 -- AppImage):**
1. Cleonaのウェブサイトまたは GitHub Releases から.AppImageファイルをダウンロードします。
2. ファイルに実行権限を与えます: 右クリック → プロパティ → 実行可能にする、
   またはターミナルで: `chmod +x cleona-chat-VERSION-x86_64.AppImage`
3. ダブルクリックするか、ターミナルで次を実行して起動します: `./cleona-chat-VERSION-x86_64.AppImage`

**Windows:**
1. Cleonaのウェブサイトまたは GitHub Releases からインストーラーをダウンロードします。
2. インストールファイルを実行し、案内に従って進めます。
3. スタートメニューまたはデスクトップのショートカットからCleonaを起動します。

### アイデンティティの作成

初回起動時、Cleonaは自動的に新しいアイデンティティを作成します。表示名
（連絡先に表示される名前）を自由に設定できます。この名前はいつでも
変更可能です。

### シードフレーズを書き留める -- 最も重要なこと

アイデンティティを作成した後、Cleonaは24個の単語を表示します。これがあなたの
**シードフレーズ** -- 個人用の復元キーです。

**この24個の単語を紙に書き留め、安全な場所に保管してください。**

なぜこれがそれほど重要なのでしょうか？

- スマートフォンが壊れたり、紛失したり、盗まれたりした場合でも、この24個の
  単語を使えば新しい端末であなたのアイデンティティ全体を復元できます。
- シードフレーズがなければ、後戻りする方法はありません。「パスワードをお忘れ
  ですか」ボタンも、アカウントを取り戻してくれるサポートもありません --
  そもそもサーバー上にアカウント自体が存在しないからです。
- シードフレーズを決して他人に教えないでください。この単語を知っている人は、
  あなたになりすますことができます。

シードフレーズは後から設定の「セキュリティ」でも確認できます。もう一度
見たいときはそちらを利用してください。

### 最初の連絡先を追加する

誰かとチャットするには、まずその相手を連絡先として追加する必要があります。
その方法はいくつかあり、次のセクションですべて説明します。

---

## 3. 連絡先

### QRコードをスキャンする（推奨）

連絡先を追加する最も簡単な方法です。

1. 相手が自分のアイデンティティ詳細ページを開き（上部バーの自分の名前を
   タップ）、QRコードを表示します。
2. あなたはプラスボタンをタップし、「QRコードをスキャン」を選択します。
3. スマートフォンを相手のQRコードにかざします。
4. 連絡先リクエストが自動的に送信されます。相手が承認すると、お互いに
   メッセージのやり取りができるようになります。

直接会って行う場合、QRコードは誰と連絡先を交換しているかを正確に把握できる
ため、最も安全な方法です。

### NFC（スマートフォン同士をかざす）

両方の端末がNFCに対応している場合:

1. 両方の端末で「連絡先を追加」機能を開きます。
2. スマートフォンの背面同士を合わせます。
3. 連絡先情報が自動的に交換されます。

NFCはQRコードと同様に高い安全性を提供します。物理的に隣り合っている場合
にのみ交換が成立するためです。

### リンクを共有する（cleona://URI）

連絡先リンクをメール、SMS、または他のメッセンジャー経由で送ることもできます。

1. 自分のアイデンティティ詳細ページを開きます。
2. cleona://リンクをコピーします。
3. 追加してほしい相手にリンクを送信します。
4. 相手はそのリンクを開くか、「連絡先を追加」ダイアログに貼り付けます。

注意: この方法では、伝送経路上でリンクが改ざんされていないことを信頼する
ことになります。特に慎重を要する連絡先には、QRコードまたはNFCを
お勧めします。

### 連絡先リクエストを承認する

誰かがあなたに連絡先リクエストを送ると、受信箱（下部バーの一番右のタブ）に
表示されます。そこで次の操作ができます。

- **承認** -- 相手があなたの連絡先に追加されます。
- **拒否** -- リクエストは破棄されます。
- **ブロック** -- 相手はこれ以上リクエストを送れなくなります。

### 検証レベル

Cleonaは、連絡先のアイデンティティがどの程度確実に確認されているかを
表示します。

| レベル | 意味 |
|-------|-----------|
| 不明 | ノードIDまたはリンクを受け取っただけの状態です。 |
| 確認済み | 鍵交換が成功し、暗号化された通信ができる状態です。 |
| 検証済み | 直接会い、QRコードまたはNFCで検証した状態です。 |
| 信頼済み | この連絡先を明示的に信頼できるものとしてマークした状態です。 |

レベルが高いほど、本当に正しい相手と話していると確信できます。

## 4. メッセージ

### テキストの送受信

下部の入力欄にメッセージを入力し、Enterキーまたは送信ボタンを押すだけです。
メッセージは端末を離れる前に自動的に暗号化されます。

受信したメッセージはチャット履歴に表示されます。チェックマークで、
メッセージが配達されたかどうかを確認できます。

### 画像、動画、ファイルの送信

いくつかの方法があります。

- **入力欄のクリップアイコン**: タップすると、ギャラリーやファイルシステム
  からファイル、画像、動画を選択できます。
- **ドラッグ＆ドロップ**（デスクトップ）: ファイルをチャットウィンドウに
  ドラッグするだけです。
- **クリップボードから貼り付け**（デスクトップ）: 画像をコピーして
  チャットに貼り付けます。

小さいファイル（256KB未満）はそのまま直接送信されます。より大きな
ファイルは二段階の手順で転送されます。まずファイルの通知が送られ、
その後に分割して送信されます。

### 音声メッセージ

1. 入力欄のマイクボタンを押し続けます。
2. メッセージを話します。
3. ボタンを離すとメッセージが送信されます。

端末で音声認識が有効になっている場合（設定を参照）、音声メッセージは
自動的にテキストへ書き起こされます。相手には録音とテキストの両方が
表示されます。

### メッセージへの返信（引用）

特定のメッセージに返信するには:

1. メッセージ横の3点メニューを開きます。
2. 「返信」を選択します。
3. 入力欄の上に、引用したメッセージを示すバナーが表示されます。
4. 返信を書いて送信します。

引用されたメッセージは返信内に表示されるため、どのメッセージへの返信か
明確になります。

### メッセージの編集と削除

- **編集:** メッセージの3点メニュー、「編集」を選択します。テキストを
  変更して再送信します。相手にはメッセージが編集されたことが表示されます。
  編集は送信から15分以内に可能です。
- **削除:** メッセージの3点メニュー、「削除」を選択します。メッセージは
  あなたと相手の両方から削除されます。自分のメッセージはいつでも削除
  できます -- 削除に時間制限はありません。

### 絵文字リアクション

返信を書く代わりに、メッセージに絵文字でリアクションできます。

1. 3点メニューを開くか、メッセージを長押しします。
2. クイック選択から絵文字を選ぶか、絵文字ピッカーを開いて全ての選択肢
   から選びます。
3. あなたのリアクションがメッセージの下に表示されます。

### テキストのコピー

メッセージの3点メニューから、メッセージのテキストをクリップボードに
コピーできます。

### メッセージの検索

チャットウィンドウの上部に検索機能があります。検索語を入力すると、
Cleonaは現在のチャット内のすべての一致箇所を表示します。矢印キーで
一致箇所の間を移動できます。

ホーム画面には、さらにタブを横断した検索フィルターがあり、すべての
会話をキーワードで検索できます。

### リンクプレビュー

リンクを送信すると、Cleonaは自動的にプレビュー（タイトル、説明、
プレビュー画像）を生成します。このプレビューはあなたの端末で作成され、
一緒に送信されます -- 相手はリンク先のウェブサイトに接続する必要が
ありません。

受信したリンクをタップすると、通常のブラウザで開くか、シークレット
モードで開くか、あるいは開かないかを選ぶよう確認されます。

## 5. グループ

### グループの作成

1. 「グループ」タブに切り替えます。
2. プラスボタンをタップします。
3. グループに名前を付けます。
4. 招待したい連絡先を選択します。
5. 「作成」をタップします。

招待された連絡先には通知が届き、グループに参加できます。

### メンバーの招待

作成後もさらに連絡先を招待できます。

1. グループ情報を開きます（グループ一覧の3点メニュー、またはグループ
   チャット上部バー）。
2. 「招待」をタップします。
3. 追加したい連絡先を選択します。

### 役割

各グループには3つの役割があります。

- **オーナー（Owner）:** 全ての権限を持ちます。メンバーの追加・削除、
  管理者の任命、グループの管理ができます。オーナーは自分の権限を
  他のメンバーに譲渡することもできます。
- **管理者（Admin）:** メンバーを削除し、管理を手伝うことができます。
- **メンバー（Member）:** メッセージの閲覧と送信ができます。

### グループを退出する

1. グループ一覧の3点メニューを開きます。
2. 「退出」を選択します。
3. 決定を確定します。

グループを退出しても、それまでのあなたのメッセージは他のメンバーに
引き続き表示されます。

## 6. 公開チャンネル

### チャンネルとは？

チャンネルはCleonaネットワーク内の公開討論フォーラムです。グループとは
異なり、招待されなくても誰でも閲覧できます。投稿できるのはオーナーと
管理者のみで、購読者は閲覧するだけです。

### チャンネルを探して参加する

1. 「チャンネル」タブに切り替えます。
2. 「検索」タブを開きます。
3. 名前やテーマで利用可能なチャンネルを検索します。
4. チャンネルをタップし、「購読」をタップします。

チャンネルは言語でフィルタリングできます。一部のチャンネルは「年齢制限
あり」と表示されます -- これらはプロフィールで18歳以上であることを
確認した場合にのみ表示されます。

### 自分のチャンネルを作成する

1. 「チャンネル」タブに切り替えます。
2. プラスボタンをタップします。
3. チャンネル名を入力します（ネットワーク全体で一意である必要があります）。
4. 言語と、チャンネルを公開にするか非公開にするかを選択します。
5. 任意で説明と画像を追加します。
6. 「作成」をタップします。

公開チャンネルでは、コンテンツを「年齢制限あり」として分類するかどうかを
設定できます。

### コンテンツを報告する

公開チャンネルで不適切なコンテンツに気づいた場合、報告することができます。
Cleonaは分散型のモデレーションシステムを採用しています。報告は
ネットワークからランダムに選ばれたメンバーによって評価されます
（一種の「陪審制度」）。違反が認められると、そのチャンネルには警告が
出されます。違反が繰り返されると、検索インデックスで順位が下げられるか、
封鎖されます。

### システムチャンネル

Cleonaには2つの組み込みシステムチャンネルがあります。

- **バグログ:** Cleonaがエラーを検知すると、匿名化されたエラーレポートを
  送信するかどうか確認します。これらのレポートはバグログチャンネルに
  投稿され、コミュニティが閲覧できます。個人情報は一切送信されません --
  技術的なエラー内容のみです。手動でログレポートを送信することもできます
  （プレビューダイアログと明示的な同意付き）。
- **機能リクエスト:** ユーザーが機能の要望を提出したり、既存の提案に
  投票したりできる場所です。提案は投票数順に並べられます。

両方のシステムチャンネルにはサイズ上限（25MB）があり、陪審モデレーション
システムによって監視されています。

## 7. 通話

### 音声通話を開始する

1. 通話したい連絡先とのチャットを開きます。
2. 上部バーの電話アイコンをタップします。
3. 相手が通話に応答するのを待ちます。

通話中は、通話時間を示すタイムラインが表示され、ミュートとスピーカーの
操作ができます。

通話を終了するには、赤い終了ボタンをタップします。

### ビデオ通話を開始する

1. 連絡先とのチャットを開きます。
2. 上部バーのカメラアイコンをタップします。
3. あなたの映像は小さいウィンドウに、相手の映像は大きな領域に表示
   されます。

通話中にフロントカメラとリアカメラを切り替えることができます。

### 着信

誰かがあなたに電話をかけると、発信者名を表示する通知画面が現れます。
以下の操作ができます。

- **応答** -- 通話が始まります。
- **拒否** -- 発信者に通知されます。

すでに通話中の場合、新しい着信は自動的に拒否されます。

### グループ通話

複数の人が同時に参加するグループ通話も行えます。通話はインテリジェントな
中継ツリーを通じて構成されるため、すべての参加者が互いに直接接続される
必要はありません。すべての通話はエンドツーエンドで暗号化されます。

### 通話の暗号化

すべての通話は、通話時間中だけ存在する使い捨ての鍵で暗号化されます。
通話終了後、これらの鍵は即座に削除されます。過去の通話を後から
復号できる人は誰もいません。

## 8. カレンダー

Cleonaには、暗号化され完全に分散型で動作する組み込みカレンダーが
搭載されています -- クラウドサービス不要です。

### 表示形式

カレンダーには5つの表示形式があります: 日、週、月、年、そしてタスク
表示です。カレンダー画面の上部にあるタブで切り替えられます。

### 予定の作成

タイムスロットをタップするか、追加ボタンを使って新しい予定を作成します。
タイトル、日付、時刻、場所、メモを入力できます。予定は端末上に暗号化
されて保存されます。

### 繰り返しの予定

予定は毎日、毎週、毎月、または毎年繰り返すように設定できます。パターン
（例: 第2火曜日ごと、毎月1日など）をカスタマイズし、終了日または
繰り返し回数を設定できます。

### 連絡先を招待する

予定の作成・編集時に、Cleonaの連絡先を招待できます。招待された相手は
暗号化されたカレンダー招待を受け取り、参加・不参加・未定で回答できます。
予定の変更は自動的にすべての招待者に送信されます。

### 空き時間/予定ありの表示

予定の詳細を明かすことなく、連絡先と自分の空き状況を共有できます。
プライバシーレベルは3段階あります: 詳細をすべて表示、時間帯のみ表示、
非表示です。デフォルトを設定し、連絡先ごとに上書きすることもできます。

### リマインダー

予定にはリマインダーを設定でき、予定開始前にシステム通知が表示されます。
必要に応じてリマインダーをスヌーズできます。

### 外部カレンダー同期

Cleonaは外部のカレンダーサービスと同期できます。

- **CalDAV** -- CalDAV対応の任意のサーバー（Nextcloud、Radicaleなど）に
  接続します。
- **Googleカレンダー** -- 安全なOAuth2認証を使ったGoogle Calendar API
  経由の同期です。
- **ローカルCalDAVサーバー** -- Cleonaは端末上にローカルCalDAVサーバーを
  起動でき、デスクトップのカレンダーアプリ（Thunderbird、Outlook、
  Appleカレンダー、Evolution）がCleonaのカレンダーと同期できます。
- **Androidシステムカレンダー** -- Cleonaの予定をAndroid端末の組み込み
  カレンダーアプリに転送できます。
- **ICSファイル** -- 標準のiCalendar形式で予定をインポート・エクスポート
  できます。

### PDFエクスポート

各カレンダー表示（日、週、月、年）をPDF文書として印刷またはエクスポート
できます。

## 9. アンケート

各チャットやグループでアンケートを作成し、意見を集めたり日程を調整したり
できます。

### アンケートの種類

Cleonaは5種類のアンケートに対応しています。

- **単一選択** -- 参加者が1つの選択肢を選びます。
- **複数選択** -- 参加者が複数の選択肢を選べます。
- **日程アンケート** -- 全員に都合の良い日程を探します。参加者はそれぞれ
  日程を「参加可能」「未定」「参加不可」としてマークします。
- **スケール** -- 数値スケール（例: 1〜5）で何かを評価します。
- **自由記述** -- 参加者が自分の回答を自由に書きます。

### アンケートの作成

チャットを開き、アンケートアイコンをタップします（または添付メニューを
使用します）。アンケートの種類を選び、質問と選択肢を入力して送信します。
チャット内にメッセージとして表示されます。

### 投票する

アンケートをタップして投票します。投票はいつでも変更または取り消す
ことができます。

### 匿名投票

アンケートは匿名投票用に設定できます。有効にすると、投票は暗号学的に
匿名化されます -- アンケート作成者を含め、誰が何に投票したかは誰にも
わかりません。それでも投票数自体は表示されます。

### 日程アンケートからカレンダーへ

日程アンケートが締め切られると、当選した日程をワンタップでそのまま
カレンダーの予定に変換できます。

## 10. 複数のアイデンティティ

### なぜ複数のアイデンティティが必要なのか

仕事とプライベートを分けたいと想像してください -- まるで2つの電話番号を
使い分けるように、しかし2台目のスマートフォンを持つ必要はありません。
Cleonaでは、1台の端末で複数のアイデンティティを利用できます。それぞれの
アイデンティティは独自の名前、プロフィール画像、連絡先、会話を持ちます。

### 新しいアイデンティティの作成

1. 上部バーには現在のアイデンティティがタブとして表示されています。
2. アイデンティティタブの右側にあるプラス記号（+）をタップします。
3. 新しいアイデンティティの名前を入力します。
4. 完了です -- 新しいアイデンティティはすぐに有効になります。

### アイデンティティの切り替え

上部バーのアイデンティティタブをタップするだけです。切り替えは即座に
行われます -- 待ち時間も再読み込みもありません。

### すべて同時に稼働する

重要な点として、すべてのアイデンティティは同時にアクティブです。現在
「仕事用」として表示されていても、「プライベート用」のアイデンティティは
引き続きメッセージを受信します。どのアイデンティティを選択していても、
何も見逃すことはありません。

### アイデンティティ詳細ページ

現在アクティブなアイデンティティのタブをタップすると、詳細ページが
開きます。ここでは以下が行えます。

- 連絡先用のQRコードを表示する。
- プロフィール画像を変更または削除する。
- プロフィールの説明文を追加する。
- 表示名を変更する。
- このアイデンティティ用のデザイン（スキン）を選択する。
- 不要になったアイデンティティを削除する。

### アイデンティティの削除

アイデンティティを削除すると、あなたの連絡先にその旨が通知されます。
アイデンティティとそれに関連するすべてのデータは端末から削除されます。
この操作は元に戻せません。

## 11. マルチデバイス

### Cleonaを複数の端末で使う

同じアイデンティティを最大5台の端末で同時に利用できます。1台がプライマリ
端末（シードフレーズを保持します）となり、他の端末はそれに紐付けられます。

### 新しい端末を紐付ける

1. プライマリ端末で設定を開きます。
2. 「連携デバイス」に移動します。
3. 「新しいデバイスを連携」を選択します。
4. 新しい端末にCleonaをインストールし、起動時に「既存のデバイスと連携」
   を選択します。
5. プライマリ端末に表示されるペアリング用QRコードをスキャンするか、
   ペアリングリンクを使用します。

紐付けられた端末は、プライマリ端末から委任証明書を受け取ります。紐付けた
端末から送信されたメッセージは、委任された鍵で暗号学的に署名されるため、
連絡先はそのメッセージが本当にあなたのアイデンティティから送られたものか
どうかを確認できます。

### 仕組み

- プライマリ端末はシードフレーズとマスター鍵を保持します。
- 紐付けられた端末は派生した署名鍵と委任証明書を受け取ります -- シード
  フレーズ自体を受け取ることは決してありません。
- すべての端末は同じアイデンティティと連絡先を共有します。メッセージは
  すべての端末に届きます。
- 委任証明書は期限が切れる前に自動的に更新されます。

### デバイス管理

設定を開き、「連携デバイス」に移動すると、紐付けられたすべての端末、
その状態、最終アクティビティを確認できます。端末を紛失したり盗まれたり
した場合、いつでも紐付けを取り消すことができます。

### 緊急鍵ローテーション

端末が侵害された疑いがある場合、緊急鍵ローテーションを実行できます。
この際、新しい鍵が生成され、そのローテーションはあなたの他の端末の
過半数によって承認される必要があります。これにより、盗まれた端末
1台だけで勝手に鍵をローテーションすることを防ぎます。

## 12. 復元

### シードフレーズを使う

端末を紛失した場合や、新しい端末をセットアップする場合:

1. 新しい端末にCleonaをインストールします。
2. 起動時に「復元」を選択します。
3. 24個の単語を入力します。
4. Cleonaはあなたのアイデンティティを復元し、自動的に以前の連絡先に
   連絡します。
5. 連絡先が、あなたの連絡先情報、グループメンバーシップ、メッセージ
   履歴を返信してくれます。

復元は3つの段階で行われます。
- まず連絡先とグループが戻ってきます。
- 次に各会話の直近50件のメッセージが戻ります。
- 最後に完全なメッセージ履歴が戻ります。

復元が機能するためには、連絡先のうちたった1人がオンラインであれば
十分です。

### ガーディアン・リカバリー（信頼できる人）

最大5人の信頼できる人物を「ガーディアン」として指定できます。この際、
あなたの復元キーは5つの部分に分割され、各ガーディアンがそのうちの1つを
受け取ります。アイデンティティを復元するには、5つのうち3つの部分が
あれば十分です。

つまり、シードフレーズを紛失してしまった場合でも、3人のガーディアンが
協力すればあなたのアカウントを復元できるということです。単独の
ガーディアンがあなたのデータに単独でアクセスすることはできません --
常に最低3人が必要です。

ガーディアンの設定方法:
1. 設定を開きます。
2. 「セキュリティ」に移動します。
3. 「ガーディアン・リカバリー」を選択します。
4. 信頼できる連絡先を5人選択します。

### なぜ連絡先があなたのバックアップなのか

従来のメッセンジャーでは、あなたのデータは提供元のサーバー上に保存されて
います。Cleonaにはサーバーが存在しませんが -- その代わりにあなたの連絡先が
この役割を担います。メッセージを送信すると、共通の連絡先が受信者が
オフラインの場合に備えて暗号化されたコピーを保存します。復元の際には、
連絡先があなたにデータを返してくれます。

つまり、アクティブな連絡先が多いほど、あなたのバックアップはより
信頼できるものになります。定期的にオンラインになる連絡先が1人いれば、
復元は成功します。

## 13. 設定

設定には右上の歯車アイコンからアクセスできます。

### 通知と着信音

- 着信用に6種類の着信音から選べます。
- メッセージ通知音を設定します。
- Android端末では、さらにバイブレーションのオン・オフを切り替えられます。

### デザイン（スキン）

Cleonaは10種類のデザインを提供しています: Teal、Ocean、Sunset、Forest、
Amethyst、Fire、Storm、Slate、Gold、Contrastです。Contrastデザインは
最高水準のアクセシビリティ基準（WCAG AAA）を満たしており、視力に制約が
ある場合でも特に読みやすくなっています。

各アイデンティティは独自のデザインを持つことができます。デザインの
変更はアイデンティティ詳細ページ（アクティブなアイデンティティタブを
タップ）で行います。

さらに、設定の「外観」で、ライト、ダーク、システムテーマの間を
切り替えられます。

### 言語の変更

Cleonaは33言語に対応しており、右から左に書く言語（アラビア語、
ヘブライ語など）も含まれます。設定の「言語」で言語を変更できます。

### ストレージ上限

Cleonaが端末上で使用できるストレージ容量（100MBから2GBの間）を設定
できます。上限に達すると、古いメディアは自動的に外部保存されるか
削除されます -- テキストメッセージは常に保持されます。

### メディアアーカイブ

自宅にネットワークストレージ（NAS）や共有フォルダがある場合、Cleonaは
メディアを自動的にそこへ移すことができます。SMB、SFTP、FTPS、WebDAVに
対応しています。

段階的な保存の仕組みは次の通りです。

- 最初の30日間: すべて端末上に残ります。
- 30日後: プレビュー画像が端末上に残り、オリジナルはアーカイブされます。
- 90日後: 小さなプレビュー画像のみが端末上に残ります。
- 1年後: プレースホルダーのみが残り、オリジナルはアーカイブに安全に
  保管されます。

アーカイブされたメディアはいつでもタップして取り戻すことができます --
ただし自宅のネットワークに接続している必要があります。特に重要な
メディアはピン留めして、決して外部保存されないようにすることも
できます。

### 音声メッセージの文字起こし

有効にすると、音声メッセージは端末上でローカルにテキストへ変換されます
（オープンソースモデルWhisperを使用）。書き起こされたテキストは録音と
一緒に相手に送信されます。文字起こしは完全に端末上で行われます --
外部サービスにデータが送られることはありません。

### 自動ダウンロード

メディアを自動的にダウンロードするサイズの基準を設定できます。例えば
画像は自動的に読み込ませつつ、大きな動画については手動で判断する、
といった使い方ができます。

### 連携デバイス

この設定項目で、紐付けられたデバイスを管理します。詳細はマルチデバイス
の章を参照してください。

## 14. セキュリティ

### ポスト量子暗号とは？

現在の暗号化は、通常のコンピューターでは解くのが極めて困難な数学的問題に
基づいています。量子コンピューターは将来、こうした問題の一部を高速に
解けるようになる可能性があります。ポスト量子暗号は、量子コンピューターに
対しても耐性のある追加の手法を用います。

Cleonaは両方のアプローチを組み合わせています: 信頼性のための古典的暗号化と、
将来の安全性のためのポスト量子技術です。これにより、現在と将来の脅威の
両方から同時に守られます。

メッセージ1件ごとに専用の鍵が生成されます。仮に攻撃者がある1件のメッセージ
の鍵を解読できたとしても、それによって他のメッセージを読むことは
できません。

### サーバーがないことがなぜより安全なのか

従来のメッセンジャーでは、メッセージは提供元のサーバーを経由します。
たとえそこで暗号化されていたとしても、提供元はメタデータ（誰がいつ誰と
どのくらいの頻度で、どこから通信しているか）にアクセスでき、場合によっては
裁判所の命令によりそれを開示しなければなりません。

Cleonaにはそのような中央集約点は存在しません。あなたのメッセージは端末から
端末へ直接移動します。すべてのメタデータが集まる場所はどこにもありません。
たった1つのデータポイントから、あなたのコミュニケーション行動を
再構築できる人は誰もいません。

### オフラインのときはどうなるのか

メッセージを送信し、受信者がオフラインの場合:

1. Cleonaはまずメッセージを直接配達しようとします。
2. うまくいかない場合は、共通の連絡先を経由して転送されます。
3. 同時に、メッセージは暗号化された断片としてネットワーク内の複数の
   ノードに分散されます（10個のピースからなるパズルのようなもので、
   そのうち7個があれば絵を組み立てられる、といったイメージです）。
4. メッセージは最大7日間保管されます。

受信者が再びオンラインになると、メッセージが配達されます。メッセージが
届くと、あなたには配達確認が届きます。

### 検閲耐性

あなたのネットワークが標準の接続方式（UDP）をブロックしている場合、
Cleonaは自動的に、検出やブロックがより困難な代替の伝送方式（TLS）に
切り替わります。これは透過的に行われます -- 設定を行う必要はありません。

### 安全な鍵の保管

対応プラットフォームでは、Cleonaは暗号鍵をオペレーティングシステムの
安全なキーリング（Androidキーストア、iOSキーチェーン、macOSキーチェーン）
に保存します。利用可能な場合は、鍵に対してハードウェアによる保護が
提供されます。

### データベースの暗号化

あなたのメッセージ、連絡先、設定はすべて端末上で暗号化されて保存されて
います。仮に誰かがファイルシステムにアクセスできたとしても、あなたの
暗号鍵がなければ何も読み取ることはできません。この鍵はあなたの
アイデンティティから導出され、あなたの端末上にのみ存在します。

### 閉じたネットワーク

Cleonaは閉じたネットワークとして動作します。すべてのネットワークパケット
は認証されており、正規のCleona端末のみが参加できます。これにより、
部外者が偽のメッセージを紛れ込ませたり、ネットワークの通信を盗聴したり
することを防ぎます。

## 15. ソフトウェアアップデート

### アップデートはどのように入手しますか？

Cleonaはさまざまな方法でアップデートできます。目的は、一部の配布経路が
使えなくなったり遮断されたりした場合でも、アップデートを受け取り続け
られるようにすることです。

1. **App Store / Play Store:** App Store経由でCleonaをインストールした
   場合、通常どおりストアを通じてアップデートを受け取れます。
2. **GitHub Releases:** プロジェクトのGitHubページで、全プラットフォーム
   向けの署名済みインストールパッケージが見つかります。
3. **ネットワーク内アップデート:** あなたのネットワーク内の他のCleona
   ユーザーがすでに最新バージョンを持っている場合、Cleonaは外部サーバー
   を使わずにP2Pネットワーク経由で直接アップデートを取得できます。この際、
   新しいバージョンは誤り訂正符号付きの断片に分割され、複数のノードに
   分散されます。あなたの端末は十分な数の断片を集め、アップデートを
   組み立てます。真正性は開発者によるEd25519署名によって検証されます。
4. **招待リンク:** 新しいユーザーがCleonaをインストールしてネットワークに
   接続するために必要なものをすべて含む招待リンクを作成できます。
5. **物理的な転送:** インターネットのない環境では、USBメモリや
   ローカルネットワーク経由でCleonaを他の人に渡すことができます。

### アップデート通知

新しいアップデートが利用可能になると、Cleonaはホーム画面に通知を
表示します。アップデートがネットワーク経由でも利用可能な場合
（ネットワーク内アップデート）、ネットワークから直接ダウンロードするか
どうかを選択できます。

### バイナリ配布

デフォルトでは、あなたの端末はネットワーク内の他のユーザーへアップデート
を配布する手助けをします。これを望まない場合は、設定の「ネットワーク」で
この機能を無効にできます。アップデート断片のためのストレージ使用量には
上限があり（モバイル端末で5MB、デスクトップ端末で20MB）、定期的に
整理されます。

### 署名の検証

すべてのアップデートは暗号学的に署名されています。Cleonaはアップデートを
インストールする前に自動的に署名を検証します。これにより、P2Pネットワーク
経由で取得したアップデートであっても、正規の開発者によるアップデートのみが
受け入れられることが保証されます。

## 16. よくある質問

### 「インターネットなしでCleonaを使えますか？」

いいえ、Cleonaはメッセージを送受信するためにネットワーク接続が必要です。
ただし、相手と同時にオンラインである必要はありません。受信者がオフライン
の間に送信されたメッセージは一時保存され、両者が再び接続されると自動的に
配達されます。ローカルネットワーク内（例えば同じWLAN内）であれば、
インターネット接続なしでも互いに通信できます。

### 「シードフレーズを紛失したらどうなりますか？」

ガーディアンを設定している場合、5人のうち3人の信頼できる人物が協力して
あなたのアクセスを復元できます。ガーディアンもシードフレーズもない場合、
残念ながらアイデンティティを取り戻す方法はありません。だからこそ、24個の
単語を安全に保管しておくことがそれほど重要なのです。

### 「誰かが私のメッセージを盗み見ることはできますか？」

いいえ。すべてのメッセージは、その1件のためだけに使われる使い捨ての鍵で
暗号化されます。メッセージを復号できるのはあなたと相手だけです。中央
サーバーもマスターキーも、開発者用のアクセス経路も存在しません。仮に
転送経路上のある端末がメッセージを中継したとしても、そこには暗号化された
データの塊しか見えません。

### 「なぜ電話番号が不要なのですか？」

あなたのアイデンティティが純粋に暗号技術に基づいているからです。実名に
紐付いた電話番号やメールアドレスの代わりに、端末上で生成された鍵ペアが
あなたを識別します。連絡先はQRコード、NFC、リンクを通じて追加します --
電話帳経由ではありません。これは、あなたのメッセンジャーアカウントが
現実のアイデンティティに結び付けられていないため、より高いプライバシーを
意味します。

### 「Cleonaで人を見つけるにはどうすればいいですか？」

Cleonaには意図的に、電話番号や名前による連絡先検索機能がありません --
それはプライバシー上の問題になるためです。代わりに、連絡先情報を直接
交換します: QRコード、NFC、cleona://リンク、または公開チャンネルを
通じて。これは電話帳で調べる代わりに名刺を交換するようなものです。

### 「Cleonaは海外でも使えますか？」

はい。インターネット接続さえあれば、Cleonaは世界中どこでも動作します。
中央サーバーが存在しないため、特定の国向けにサービスを遮断することも
できません。さらにCleonaには検閲耐性のフォールバック機能があります:
通常の接続方式（UDP）がブロックされた場合、Cleonaは自動的に、検出や
ブロックがより困難な代替の伝送方式（TLS）に切り替わります。

### 「Cleonaは無料ですか？」

はい。Cleonaは無料で、広告なしで利用できます。中央サーバーが存在しない
ため、運用のためのサーバー費用も発生しません。アプリ内の「寄付」から、
開発を任意で支援することができます。

### 「メッセージに時計のマークが付いています -- どういう意味ですか？」

これは、メッセージがまだ配達されていないことを意味します。相手はおそらく
現在オフラインです。メッセージが配達されると、マークが変わります。
メッセージは配達のために最大7日間保管されます。

### 「WhatsAppからCleonaに乗り換えることはできますか？」

はい、ただしWhatsAppのチャットを引き継ぐことはできません。CleonaとWhatsApp
はまったく異なるシステムです。連絡先は1人ずつCleonaに追加する必要が
あります。最も簡単な方法は、cleona://リンクをWhatsAppのグループに投稿し、
他のメンバーにそこであなたを追加してもらうよう頼むことです。

### 「Cleonaを複数の端末で同時に使えますか？」

はい。同じアイデンティティで最大5台の端末を紐付けることができます。1台が
プライマリ端末（シードフレーズを保持します）となり、他の端末は安全な
ペアリングプロセスを通じて紐付けられます。すべての端末は同じアイデンティティ、
連絡先、会話を共有します。詳細はマルチデバイスの章を参照してください。

### 「App Storeがブロックされている場合、どうやってアップデートを入手すればいいですか？」

CleonaはApp Store、ウェブサイト、ダウンロードサーバーに依存せず、P2P
ネットワーク経由で直接アップデートを取得できます。ネットワーク内の他の
ユーザーが最新バージョンを持っている場合、あなたの端末はそこからアップデート
を読み込めます。真正性は開発者によるデジタル署名で検証されます。あるいは、
連絡先が招待リンクやUSBメモリでアプリを渡してくれることもできます。詳しくは
「ソフトウェアアップデート」の章を参照してください。
