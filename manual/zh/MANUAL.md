## 1. 什么是 Cleona Chat？

### 你的即时通讯工具,你的数据

Cleona Chat 是一款完全不依赖中心服务器运行的即时通讯软件。你的消息直接从你的设备传送到对方的设备——不绕道任何公司总部,不经过云端,不经过数据中心。没有任何公司能够读取、存储或转发你的消息,因为压根就没有公司置身其中。

### 无需账号,无需手机号

使用 Cleona 时,你既不需要手机号,也不需要电子邮箱来注册登录。你的身份由一对加密密钥组成,首次启动时会自动在你的设备上生成。这意味着:除非你自己主动分享联系方式,否则没有人能通过手机号或电子邮箱找到你。

### 面向未来的加密技术

Cleona 采用所谓的后量子加密技术。这意味着:即使是未来的量子计算机,也无法破解你的消息。你不必了解其中的技术细节——重要的是,你的通讯已经按照当前技术水平得到了最佳保护。

### 没有服务器,这是如何做到的?

设想一下,你和你的联系人共同组成了一张网络。每一台设备都在帮助转发消息。如果对方正在线上,消息会直接送达。如果对方离线,你们共同的联系人会暂存这条消息,并在收件人重新上线后转交给他。也就是说,你的联系人同时也是你的网络。

### 支持平台

Cleona 支持 Android、iOS、macOS、Linux 和 Windows。

## 2. 快速上手

### 安装应用

**Android:**
1. 从 Cleona 官网或 GitHub Releases 下载 APK 文件。
2. 在手机上打开该文件。如有需要,请允许安装来自未知来源的应用(Android 会自动提示你)。
3. 点击"安装",等待安装完成。

**iOS:**
1. 在 iPhone 上打开 TestFlight 邀请链接。
2. 点击"安装"。TestFlight 是 Apple 官方用于分发测试版应用的渠道。
3. 安装完成后,你可以在主屏幕上找到 Cleona。

**macOS:**
1. 从 Cleona 官网或 GitHub Releases 下载 DMG 文件。
2. 打开 DMG,将 Cleona 拖入"应用程序"文件夹。
3. 首次启动时,macOS 可能会询问你是否要打开来自已识别开发者的应用——确认即可。

**Linux(Ubuntu/Debian）：**
1. 从 Cleona 官网或 GitHub Releases 下载 .deb 文件。
2. 双击安装,或在终端中执行:`sudo dpkg -i cleona-chat_VERSION_amd64.deb`
3. 通过应用程序菜单启动 Cleona,或在终端中输入 `cleona-chat`。

**Linux(Fedora/openSUSE）：**
1. 从 Cleona 官网或 GitHub Releases 下载 .rpm 文件。
2. 使用以下命令安装:`sudo dnf install cleona-chat-VERSION.x86_64.rpm`
3. 通过应用程序菜单启动 Cleona,或在终端中输入 `cleona-chat`。

**Linux(所有发行版 —— AppImage）：**
1. 从 Cleona 官网或 GitHub Releases 下载 .AppImage 文件。
2. 为文件添加可执行权限:右键点击、选择"属性"、勾选"可执行",或在终端中执行:`chmod +x cleona-chat-VERSION-x86_64.AppImage`
3. 双击运行,或在终端中执行:`./cleona-chat-VERSION-x86_64.AppImage`

**Windows:**
1. 从 Cleona 官网或 GitHub Releases 下载安装程序。
2. 运行安装文件,并按照提示操作。
3. 通过开始菜单或桌面快捷方式启动 Cleona。

### 创建身份

首次启动时,Cleona 会自动为你创建一个新身份。你可以设置一个显示名称——这是你的联系人将会看到的名字。这个名字可以随时修改。

### 抄写助记词——最重要的一步

创建身份后,Cleona 会向你展示 24 个单词。这就是你的**助记词**——你个人专属的恢复密钥。

**请将这 24 个单词抄写在纸上,并妥善保管。**

为什么这一点如此重要?

- 如果你的手机损坏、丢失或被盗,你可以凭这 24 个单词在新设备上恢复你的完整身份。
- 没有助记词,就没有任何找回的办法。这里没有"忘记密码"按钮,也没有客服能帮你找回账号——因为服务器上根本就不存在这样的账号。
- 切勿将助记词透露给任何人。知道这些单词的人就能冒充你的身份。

如果你以后想再次查看助记词,可以在设置的"安全"部分找到它。

### 添加第一个联系人

要与某人聊天,你需要先将对方添加为联系人。添加联系人有多种方式——都会在下一节中详细说明。

## 3. 联系人

### 扫描二维码(推荐)

添加联系人最简单的方法:

1. 对方打开自己的身份详情页(点击顶部栏中自己的名字),并向你展示他的二维码。
2. 你点击加号按钮,选择"扫描二维码"。
3. 将手机对准对方的二维码。
4. 系统会自动发送联系人请求。一旦对方接受,你们就可以互相聊天了。

如果你们是当面见面,二维码是最安全的方式,因为你能清楚地知道自己是在和谁交换联系方式。

### NFC(手机贴近)

如果双方设备都支持 NFC:

1. 双方都打开"添加联系人"功能。
2. 将两台手机背对背贴近。
3. 联系方式会自动交换。

NFC 与二维码一样具有很高的安全性,因为只有当你们实际站在一起时,交换才能进行。

### 分享链接(cleona:// 链接)

你也可以通过电子邮件、短信或其他即时通讯软件发送你的联系人链接:

1. 打开你的身份详情页。
2. 复制你的 cleona:// 链接。
3. 将链接发送给想要添加你的人。
4. 对方打开该链接,或者将其粘贴到"添加联系人"对话框中。

请注意:使用这种方法时,你需要相信链接在传输过程中未被篡改。对于特别敏感的联系人,我们建议使用二维码或 NFC。

### 接受联系人请求

当有人向你发送联系人请求时,该请求会出现在你的收件箱中(底部栏的最后一个标签页)。在这里,你可以:

- **接受**——对方会被添加到你的联系人列表。
- **拒绝**——该请求会被丢弃。
- **屏蔽**——对方将无法再向你发送请求。

### 验证级别

Cleona 会向你显示某个联系人的身份得到确认的可靠程度:

| 级别 | 含义 |
|-------|-----------|
| 未知 | 你只获得了对方的节点 ID 或一个链接。 |
| 已知晓 | 密钥交换已成功完成,你们可以进行加密通讯。 |
| 已验证 | 你们已当面见过,并通过二维码或 NFC 完成了验证。 |
| 已信任 | 你已将该联系人明确标记为可信任。 |

级别越高,你就越能确信自己正在和真正的对方交流。

## 4. 消息

### 发送和接收文本消息

只需在下方的输入框中输入你的消息,然后按回车键或点击发送按钮即可。你的消息在离开设备之前会自动被加密。

收到的消息会显示在聊天记录中。一个对勾图标会告诉你消息是否已送达。

### 发送图片、视频和文件

你有多种方式可以选择:

- **输入框中的回形针图标**:点击它,从相册或文件系统中选择文件、图片或视频。
- **拖放**(桌面端):直接将文件拖入聊天窗口。
- **从剪贴板粘贴**(桌面端):复制一张图片,然后在聊天中粘贴。

较小的文件(小于 256 KB)会直接随消息一起发送。较大的文件则通过两阶段流程传输:先发送文件预告,再分块传输文件内容。

### 语音消息

1. 长按输入框中的麦克风按钮。
2. 说出你的消息内容。
3. 松开按钮即可发送消息。

如果你的设备启用了语音识别功能(见设置),你的语音消息会自动被转录为文字。对方将同时看到录音和转录出的文字。

### 回复消息(引用)

要回复某条特定消息:

1. 打开该消息旁边的三点菜单。
2. 选择"回复"。
3. 输入框上方会出现一个显示被引用消息的横幅。
4. 输入你的回复内容并发送。

被引用的消息会显示在你的回复中,让上下文关系一目了然。

### 编辑和删除消息

- **编辑:** 打开消息的三点菜单,选择"编辑"。修改文字后重新发送。对方会看到该消息已被编辑过的标记。编辑功能仅在发送后 15 分钟内可用。
- **删除:** 打开消息的三点菜单,选择"删除"。该消息会同时从你和对方的设备中移除。你可以随时删除自己发送的消息——删除没有时间限制。

### 表情回应

除了文字回复,你还可以用表情符号对一条消息作出回应:

1. 打开三点菜单,或长按该消息。
2. 从快捷选项中选择一个表情,或打开表情选择器查看完整选项。
3. 你的表情反应会显示在该消息下方。

### 复制文本

通过消息的三点菜单,你可以将消息文字复制到剪贴板。

### 搜索消息

聊天窗口顶部提供了搜索功能。输入搜索关键词,Cleona 会显示当前聊天中所有匹配的结果。你可以使用箭头按钮在结果之间跳转。

在主页上还有一个跨标签页的搜索过滤器,可用于在所有对话中按关键词搜索。

### 链接预览

当你发送一个链接时,Cleona 会自动生成预览(标题、描述、缩略图)。这个预览由你的设备生成并随消息一起发送——对方无需连接到该链接指向的网站。

当你点击收到的链接时,系统会询问你是要在普通浏览器中打开、在隐身模式中打开,还是不打开。

## 5. 群组

### 创建群组

1. 切换到"群组"标签页。
2. 点击加号按钮。
3. 为群组命名。
4. 选择你想邀请的联系人。
5. 点击"创建"。

受邀的联系人会收到通知,并可以加入该群组。

### 邀请成员

群组创建后,你仍然可以邀请更多联系人:

1. 打开群组信息(在群组概览中点击三点菜单,或在群组聊天的顶部栏中打开)。
2. 点击"邀请"。
3. 选择你想添加的联系人。

### 角色

每个群组都有三种角色:

- **所有者(Owner):** 拥有完全控制权。可以添加或移除成员、任命管理员并管理整个群组。所有者也可以将该身份转让给其他成员。
- **管理员(Admin):** 可以移除成员,并协助管理群组。
- **成员(Member):** 可以读取和发送消息。

### 退出群组

1. 在群组概览中打开三点菜单。
2. 选择"退出"。
3. 确认你的操作。

当你退出群组后,你之前发送的消息对其他成员依然可见。

## 6. 公共频道

### 什么是频道?

频道是 Cleona 网络内部的公共讨论区。与群组不同的是,任何人都可以在此阅读内容,而无需受邀。只有所有者和管理员可以发布内容——订阅者只能阅读。

### 查找并加入频道

1. 切换到"频道"标签页。
2. 打开"搜索"选项卡。
3. 按名称或主题搜索可用的频道。
4. 点击某个频道,然后点击"订阅"。

频道可以按语言进行筛选。部分频道被标记为"限制级内容"——只有在你的个人资料中确认自己已满 18 岁后,这些频道才会显示。

### 创建自己的频道

1. 切换到"频道"标签页。
2. 点击加号按钮。
3. 输入频道名称(必须在整个网络中唯一)。
4. 选择语言,并设定频道是公开还是私密。
5. 可选:添加简介和封面图片。
6. 点击"创建"。

对于公开频道,你可以设置内容是否被标记为"限制级内容"。

### 举报内容

如果你在公开频道中发现不当内容,可以对其进行举报。Cleona 采用去中心化的内容审核系统:举报会由网络中随机抽取的成员进行评审(类似一种"陪审团"机制)。如果确认存在违规,该频道会收到警告。如果多次违规,该频道会在搜索索引中被降权,或被彻底封禁。

### 系统频道

Cleona 内置了两个系统频道:

- **Bug Log(错误日志):** 当 Cleona 检测到错误时,会询问你是否愿意发送一份匿名化的错误报告。这些报告会汇总到 Bug Log 频道中,供社区查看。报告中不包含任何个人数据——只有技术性的错误描述。你也可以手动发送日志报告(会显示预览对话框,并需要你明确同意)。
- **Feature Requests(功能请求):** 用户可以在这里提交功能建议,并为已有的建议投票。建议会按票数排序。

这两个系统频道的容量上限均为 25 MB,并受到陪审团审核系统的监督。

## 7. 通话

### 发起语音通话

1. 打开你想拨打的联系人的聊天窗口。
2. 点击顶部栏中的电话图标。
3. 等待对方接听。

通话期间会显示计时器和通话时长,你还可以使用静音和免提功能。

点击红色的挂断按钮即可结束通话。

### 发起视频通话

1. 打开该联系人的聊天窗口。
2. 点击顶部栏中的摄像头图标。
3. 你的画面会显示在一个小窗口中,对方的画面则显示在主视图区域。

通话过程中,你可以在前置和后置摄像头之间切换。

### 来电

当有人拨打给你时,会弹出一个显示来电者姓名的通知窗口。你可以:

- **接听**——通话开始。
- **拒接**——对方会收到通知。

如果你正在通话中,新来电会被自动拒接。

### 群组通话

你也可以发起群组通话,让多人同时参与。通话通过一种智能转发树结构进行组织,因此并非每个参与者都需要与其他所有人直接建立连接。所有通话均全程加密。

### 通话加密

所有通话都使用一次性密钥加密,该密钥仅在通话期间存在。通话结束后,这些密钥会立即被删除。任何人都无法在事后解密已经结束的通话。

## 8. 日历

Cleona 内置了一个日历功能,完全加密且去中心化运行——不依赖任何云服务。

### 视图

日历提供五种视图:日、周、月、年,以及一个任务视图。通过日历界面顶部的标签页在它们之间切换。

### 创建日程

点击某个时间段,或使用添加按钮来创建新日程。你可以输入标题、日期、时间、地点和备注。日程会以加密形式保存在你的设备上。

### 重复日程

日程可以按天、周、月或年重复。你可以自定义重复规则(例如每隔一个星期二、每月第一天),并设置结束日期或重复次数。

### 邀请联系人

在创建或编辑日程时,你可以邀请你的 Cleona 联系人。他们会收到一份加密的日历邀请,可以回复接受、拒绝或待定。日程的任何更改都会自动发送给所有受邀者。

### 空闲/忙碌状态

你可以与联系人共享你的可用时间状态,而无需透露日程的具体细节。这里有三种隐私级别:完整详情、仅显示时间段、或完全隐藏。你可以设置一个默认级别,并针对每个联系人单独覆盖。

### 提醒

日程可以设置提醒,在日程开始前触发系统通知。如有需要,你可以将提醒设置为稍后提醒(打盹)。

### 外部日历同步

Cleona 可以与外部日历服务进行同步:

- **CalDAV** —— 连接任何兼容 CalDAV 的服务器(如 Nextcloud、Radicale 等)。
- **Google 日历** —— 通过 Google Calendar API 同步,使用安全的 OAuth2 身份验证。
- **本地 CalDAV 服务器** —— Cleona 可以在你的设备上启动一个本地 CalDAV 服务器,使桌面日历应用(Thunderbird、Outlook、Apple 日历、Evolution)能够与你的 Cleona 日历同步。
- **Android 系统日历** —— Cleona 中的日程可以导出到 Android 设备内置的日历应用中。
- **ICS 文件** —— 以标准 iCalendar 格式导入和导出日程。

### PDF 导出

你可以将任意日历视图(日、周、月、年)打印或导出为 PDF 文档。

## 9. 投票

你可以在任何聊天或群组中创建投票,用于征集意见或安排日程。

### 投票类型

Cleona 支持五种类型的投票:

- **单选** —— 参与者选择一个选项。
- **多选** —— 参与者可以选择多个选项。
- **日期投票** —— 找出一个所有人都合适的时间。每位参与者可以将各个时间标记为可以、也许或不可以。
- **评分量表** —— 以数字量表(如 1 到 5 分)对某项内容进行评分。
- **自由文本** —— 参与者可以填写自己的回答。

### 创建投票

打开一个聊天,点击投票图标(或使用附件菜单)。选择投票类型,填写你的问题和选项,然后发送投票。投票会作为一条消息出现在聊天中。

### 参与投票

点击一个投票,即可进行投票。你可以随时更改或撤回自己的投票。

### 匿名投票

投票可以配置为匿名模式。启用后,投票内容会通过加密方式实现匿名——没有人,即使是投票的创建者,也无法看到是谁投了哪一项。但投票的总数仍然可见。

### 将日期投票转为日历日程

日期投票结束后,只需点击一下,即可将获胜的时间直接转换为日历日程。

## 10. 多重身份

### 为什么需要多重身份?

设想一下,你想把工作生活和私人生活分开——就像使用两个不同的手机号一样,但不需要第二部手机。在 Cleona 中,你可以在一台设备上使用多个身份。每个身份都拥有自己的名字、头像、联系人和对话记录。

### 创建新身份

1. 在顶部栏中,你可以看到当前身份的标签页。
2. 点击身份标签页右侧的加号(+)。
3. 为新身份输入一个名字。
4. 完成——新身份会立即生效。

### 切换身份

只需点击顶部栏中的身份标签页。切换是即时的——无需等待,也无需重新加载。

### 所有身份同时运行

重要提示:你的所有身份都会同时处于活动状态。即使你当前显示为"工作"身份,你的"私人"身份仍会持续接收消息。无论你当前选中了哪个身份,你都不会错过任何消息。

### 身份详情页

点击当前活动身份的标签页,即可打开详情页。在这里,你可以:

- 显示供联系人扫描的二维码。
- 更改或移除你的头像。
- 添加个人简介。
- 修改你的显示名称。
- 为该身份选择主题(Skin)。
- 在不再需要该身份时将其删除。

### 删除身份

当你删除某个身份时,你的联系人会收到相应的通知。该身份及其所有相关数据都会从你的设备中移除。此操作无法撤销。

## 11. 多设备

### 在多台设备上使用 Cleona

你可以在最多 5 台设备上同时使用同一个身份。其中一台是主设备(持有助记词),其他设备则与之关联。

### 关联新设备

1. 在你的主设备上打开设置。
2. 进入"关联设备"。
3. 选择"关联新设备"。
4. 在新设备上安装 Cleona,并在启动时选择"与现有设备关联"。
5. 扫描主设备上显示的配对二维码,或使用配对链接。

关联设备会从主设备获得一份委托证书。从关联设备发送的消息会使用委托密钥进行加密签名,使联系人能够验证该消息确实来自你的身份。

### 工作原理

- 主设备持有你的助记词和主密钥。
- 关联设备获得的是派生出的签名密钥和一份委托证书——它们永远不会获得助记词本身。
- 所有设备共享同一个身份和联系人列表。消息会同步送达所有设备。
- 委托证书会在到期前自动续签。

### 设备管理

打开设置并进入"关联设备",可以查看所有已关联设备的状态和最近活动。如果某台关联设备丢失或被盗,你可以随时将其撤销。

### 紧急密钥轮换

如果你怀疑某台设备已被攻破,可以触发紧急密钥轮换。这会生成新的密钥,并且轮换操作必须得到你其他设备中多数的确认。这可以防止单独一台被盗设备擅自完成密钥轮换。

## 12. 恢复

### 使用助记词

如果你丢失了设备,或正在配置一台新设备:

1. 在新设备上安装 Cleona。
2. 在启动时选择"恢复"。
3. 输入你的 24 个单词。
4. Cleona 会恢复你的身份,并自动联系你之前的联系人。
5. 你的联系人会回传你的联系人数据、群组成员关系和消息记录。

恢复过程分为三个阶段:
- 首先恢复你的联系人和群组。
- 然后是每个对话中最近的 50 条消息。
- 最后是完整的消息历史。

只要有一位联系人在线,恢复过程就能顺利进行。

### Guardian Recovery(监护人恢复)

你可以指定最多五位可信任的人作为"监护人"(Guardians)。你的恢复密钥会被拆分成五份,每位监护人持有其中一份。只需五份中的三份,即可恢复你的身份。

也就是说:即使你丢失了助记词,你的三位监护人也可以共同帮你恢复账号。没有任何一位监护人能够单独访问你的数据——始终需要至少三位共同行动。

设置监护人的方法:
1. 打开设置。
2. 进入"安全"。
3. 选择"Guardian Recovery"。
4. 选择五位值得信赖的联系人。

### 为什么联系人是你的备份

在传统即时通讯软件中,你的数据存储在服务提供商的服务器上。而在 Cleona 中没有服务器——但你的联系人承担了这个角色。当你发送一条消息时,你们共同的联系人会保存一份加密副本,以防收件人当时处于离线状态。在恢复过程中,你的联系人会把数据重新提供给你。

也就是说:你的活跃联系人越多,你的备份就越可靠。只需一位经常在线的联系人,就足以成功完成恢复。

## 13. 设置

你可以通过右上角的齿轮图标进入设置。

### 通知与铃声

- 为来电选择六种不同铃声中的一种。
- 设置消息提示音。
- 在 Android 设备上,你还可以开启或关闭震动提醒。

### 主题(Skins)

Cleona 提供十种不同的主题:Teal、Ocean、Sunset、Forest、Amethyst、Fire、Storm、Slate、Gold 和 Contrast。Contrast 主题符合最高级别的无障碍标准(WCAG AAA),在视力受限的情况下也能保持良好的可读性。

每个身份都可以拥有自己的主题。你可以在身份详情页中更改主题(点击当前活动的身份标签页)。

此外,你还可以在设置的"外观"部分,在浅色模式、深色模式和跟随系统之间切换。

### 更改语言

Cleona 支持 33 种语言,其中也包括从右到左书写的语言(例如阿拉伯语、希伯来语)。在设置的"语言"部分中更改显示语言。

### 存储空间限制

你可以设置 Cleona 在你的设备上可使用的存储空间上限(介于 100 MB 到 2 GB 之间)。一旦达到该上限,较旧的媒体文件会被自动转移或删除——但文字消息始终会被保留。

### 媒体自动归档

如果你家中有网络存储设备(NAS)或共享文件夹,Cleona 可以自动将你的媒体文件转移到那里。支持 SMB、SFTP、FTPS 和 WebDAV。

分级存储的工作方式如下:
- 最初 30 天:所有内容都保留在你的设备上。
- 30 天后:设备上只保留一张缩略图,原始文件会被归档。
- 90 天后:设备上只保留一张更小的缩略图。
- 一年后:设备上仅保留一个占位符,原始文件安全地存放在归档中。

你可以随时点击已归档的媒体文件将其取回——前提是你已连接到家庭网络。特别重要的媒体文件可以被固定,从而永远不会被转移出设备。

### 语音消息转录

启用后,你的语音消息会在设备本地转换为文字(使用开源模型 Whisper)。转录出的文字会与录音一起发送给对方。转录过程完全在你的设备本地进行——不会有任何数据发送给外部服务。

### 自动下载

你可以设置从多大的文件体积开始,媒体文件不再自动下载。例如,你可以让图片自动下载,而对于较大的视频文件则手动决定是否下载。

### 关联设备

在设置的这个部分管理你的关联设备。详情请参阅"多设备"一章。

## 14. 安全

### 什么是后量子加密?

当前的加密技术基于一些对普通计算机而言极难求解的数学难题。未来的量子计算机可能能够快速求解其中一些难题。后量子加密技术采用了额外的算法,即便面对量子计算机也能够抵御。

Cleona 将两种方式结合起来:使用经典加密保证可靠性,同时采用后量子算法确保面向未来的安全性。这样一来,你可以同时防范现在和未来的安全威胁。

每一条消息都会生成一个独立的密钥。即使攻击者破解了某一条消息的密钥,也无法用它读取其他任何消息。

### 为什么没有服务器反而更安全

在传统即时通讯软件中,你的消息要经过服务提供商的服务器。即使这些消息在服务器上是加密的,服务提供商仍然可以访问元数据(谁在何时与谁通讯、频率如何、来自何处),并且在某些情况下必须依据法院命令将这些数据提交出去。

而在 Cleona 中,不存在这样一个中心化节点。你的消息直接从设备传送到设备。不存在任何一个汇集所有元数据的地方。没有人能够仅凭单一数据点还原出你的通讯行为模式。

### 当你离线时会发生什么?

当你发送一条消息、而收件人处于离线状态时:

1. Cleona 会首先尝试直接送达该消息。
2. 如果无法直接送达,消息会通过你们共同的联系人进行转发。
3. 与此同时,消息会被拆分成加密的片段,分散存储在网络中的多个节点上(类似一个由 10 块组成的拼图,只需其中 7 块就能拼出完整图像)。
4. 消息最多会被保留 7 天。

一旦收件人重新上线,消息就会被送达。当你的消息送达后,你会收到一个确认提示。

### 反审查机制

如果你所在的网络屏蔽了标准连接方式(UDP),Cleona 会自动切换到一种更难被识别和封锁的备用传输方式(TLS)。这个过程完全透明——你无需进行任何配置。

### 安全密钥存储

在受支持的平台上,Cleona 会将你的加密密钥存储在操作系统的安全密钥库中(Android Keystore、iOS Keychain、macOS Keychain)。在可用的情况下,这能为你的密钥提供硬件级别的保护。

### 数据库加密

你的所有消息、联系人和设置都会以加密形式存储在你的设备上。即使有人获得了你文件系统的访问权限,如果没有你的加密密钥,也无法读取任何内容。这个密钥是从你的身份派生而来,只存在于你的设备上。

### 封闭式网络

Cleona 作为一个封闭式网络运行。每一个网络数据包都经过身份验证,确保只有合法的 Cleona 设备才能加入网络。这可以防止外部人员植入伪造消息,或窃听网络流量。

## 15. 软件更新

### 如何获取更新?

Cleona 可以通过多种途径进行更新。这样设计的目的是,即使某些分发渠道失效或被封锁,你依然能够获取更新:

1. **App Store / Play Store:** 如果你是通过应用商店安装的 Cleona,你会像往常一样通过商店获取更新。
2. **GitHub Releases:** 在项目的 GitHub 页面上,你可以找到所有平台的签名安装包。
3. **网络内更新(In-Network-Updates):** 如果你网络中的其他 Cleona 用户已经拥有最新版本,Cleona 可以直接通过 P2P 网络获取更新——无需任何外部服务器。新版本会被拆分成带纠错能力的片段,分散存储在多个节点上。你的设备会收集足够数量的片段,并将更新重新组装起来。更新的真实性会通过开发者的 Ed25519 签名进行验证。
4. **邀请链接:** 你可以创建邀请链接,其中包含新用户安装 Cleona 并连接到网络所需的一切内容。
5. **物理传输:** 在没有互联网的环境中,你可以通过 U 盘或本地网络将 Cleona 分享给其他人。

### 更新提示

当有新更新可用时,Cleona 会在主屏幕上向你显示通知。如果该更新也可以通过网络获取(网络内更新),你可以选择直接从网络中下载它。

### 二进制文件分发

默认情况下,你的设备会协助将更新分发给网络中的其他用户。如果你不希望这样,可以在设置的"网络"部分关闭该功能。用于存储更新片段的空间是有限的(移动设备 5 MB,桌面设备 20 MB),并会定期清理。

### 签名验证

每一个更新都经过加密签名。在安装更新之前,Cleona 会自动验证其签名。这确保了只有来自官方开发者的更新才会被接受——即使该更新是通过 P2P 网络获取的。

## 16. 常见问题

### "我可以在没有网络的情况下使用 Cleona 吗?"

不可以,Cleona 需要网络连接才能发送和接收消息。不过,你不需要与对方同时在线:在收件人离线期间发送的消息会被暂存,并在双方重新连接后自动送达。在同一局域网内(例如同一个 WLAN),你们甚至可以在完全没有互联网接入的情况下互相通讯。

### "如果我丢失了助记词该怎么办?"

如果你设置了监护人,五位可信任的人中的三位可以共同帮你恢复访问权限。如果既没有设置监护人,又丢失了助记词,那么很遗憾,没有任何办法能找回你的身份。这正是为什么妥善保管这 24 个单词如此重要。

### "有人能窃听我的消息吗?"

不能。每一条消息都使用仅对该消息本身有效的一次性密钥进行加密。只有你和对方才能解密这条消息。这里没有中心服务器,没有万能密钥,开发者也无法访问。即使有设备在传输路径上转发这条消息,它看到的也只是一堆加密后的乱码数据。

### "为什么我不需要手机号?"

因为你的身份完全基于密码学。与真实姓名绑定的手机号或电子邮箱不同,标识你身份的是一对在你设备上生成的密钥。你通过二维码、NFC 或链接来添加联系人——而不是通过通讯录。这意味着更高的隐私性,因为你的即时通讯账号并未与你的真实身份绑定。

### "我怎样才能在 Cleona 上找到别人?"

Cleona 有意不提供按手机号或姓名搜索联系人的功能——这样做会带来隐私问题。取而代之的是,你可以直接交换联系方式:通过二维码、NFC、cleona:// 链接,或在公共频道中。这就像交换名片,而不是在通讯录里查找。

### "Cleona 在国外也能使用吗?"

可以。只要你有互联网连接,Cleona 在世界任何地方都能正常使用。由于没有中心服务器,该服务也无法针对特定国家被封锁。此外,Cleona 还配备了反审查备用机制:如果常规连接方式(UDP)被封锁,Cleona 会自动切换到一种更难被识别和封锁的备用传输方式(TLS)。

### "Cleona 是免费的吗?"

是的。Cleona 完全免费使用,且没有任何广告。由于没有中心服务器,运营也就不产生任何服务器成本。在应用中的"捐赠"部分,你可以自愿支持该项目的开发。

### "我的消息上有一个时钟图标——这是什么意思?"

这表示该消息尚未送达。对方目前可能处于离线状态。一旦消息送达,图标就会发生变化。消息会为等待送达而保留最多 7 天。

### "我可以从 WhatsApp 换用 Cleona 吗?"

可以,但你无法迁移你的 WhatsApp 聊天记录。Cleona 和 WhatsApp 是两套完全不同的系统。你需要在 Cleona 中逐一重新添加你的联系人。最简单的方法是,把你的 cleona:// 链接发到一个 WhatsApp 群组里,请其他人通过这个链接添加你。

### "我可以同时在多台设备上使用 Cleona 吗?"

可以。你可以将最多 5 台设备与同一个身份关联。其中一台是主设备(持有助记词),其他设备则通过安全的配对流程与之关联。所有设备共享相同的身份、联系人和对话记录。详情请参阅"多设备"一章。

### "如果应用商店被封锁,我要怎么获取更新?"

Cleona 可以直接通过 P2P 网络获取更新,而无需依赖任何应用商店、网站或下载服务器。如果网络中的其他用户拥有最新版本,你的设备就可以从那里获取更新。更新的真实性会通过开发者的数字签名进行验证。另外,你的联系人也可以通过邀请链接或 U 盘把应用分享给你。更多详情请参阅"软件更新"一章。
