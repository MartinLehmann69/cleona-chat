## 1. Cleona Chat란 무엇인가요?

### 나의 메신저, 나의 데이터

Cleona Chat은 중앙 서버 없이 완전히 작동하는 메신저입니다. 여러분의 메시지는
본사나 클라우드, 데이터센터를 거치지 않고 여러분의 기기에서 상대방의 기기로
직접 전달됩니다. 그 사이에 어떤 회사도 개입하지 않기 때문에, 어떤 기업도
여러분의 메시지를 읽거나 저장하거나 다른 곳에 전달할 수 없습니다.

### 계정도, 전화번호도 필요 없습니다

Cleona를 이용하려면 전화번호나 이메일 주소가 필요하지 않습니다. 여러분의
아이덴티티는 최초 실행 시 기기에서 자동으로 생성되는 암호화 키 쌍으로
구성됩니다. 즉, 여러분이 직접 연락처 정보를 공유하지 않는 한, 누구도
전화번호나 이메일 주소를 통해 여러분을 찾아낼 수 없습니다.

### 미래에도 안전한 암호화

Cleona는 이른바 포스트 퀀텀(Post-Quantum) 암호화를 사용합니다. 즉, 미래의
양자 컴퓨터로도 여러분의 메시지를 해독할 수 없다는 뜻입니다. 세부 사항을
이해할 필요는 없습니다 -- 중요한 것은 여러분의 커뮤니케이션이 현재 기술
수준에서 최대한으로 보호된다는 점입니다.

### 서버 없이 어떻게 작동하나요?

여러분과 여러분의 연락처가 함께 하나의 네트워크를 이룬다고 상상해 보십시오.
모든 기기가 메시지를 전달하는 데 도움을 줍니다. 상대방이 지금 온라인
상태라면 메시지는 곧바로 전달됩니다. 상대방이 오프라인이라면, 공통으로 아는
연락처들이 메시지를 중간에 저장해 두었다가 수신자가 다시 온라인이 되는
즉시 전달합니다. 즉, 여러분의 연락처가 곧 여러분의 네트워크이기도 합니다.

### 지원 플랫폼

Cleona는 Android, iOS, macOS, Linux, Windows에서 사용할 수 있습니다.

## 2. 시작하기

### 앱 설치하기

**Android:**
1. Cleona 웹사이트 또는 GitHub Releases에서 APK 파일을 다운로드합니다.
2. 휴대폰에서 해당 파일을 엽니다. 필요한 경우 출처를 알 수 없는 앱 설치를
   허용합니다(Android가 자동으로 물어봅니다).
3. "설치"를 탭하고 설치가 완료될 때까지 기다립니다.

**iOS:**
1. iPhone에서 TestFlight 초대 링크를 엽니다.
2. "설치"를 탭합니다. TestFlight는 Apple이 베타 앱을 배포하는 공식적인
   방법입니다.
3. 설치가 완료되면 홈 화면에서 Cleona를 찾을 수 있습니다.

**macOS:**
1. Cleona 웹사이트 또는 GitHub Releases에서 DMG 파일을 다운로드합니다.
2. DMG를 열고 Cleona를 응용 프로그램 폴더로 드래그합니다.
3. 처음 실행할 때 macOS가 신원이 확인된 개발자의 앱을 열 것인지 물어볼 수
   있습니다 -- 이를 확인해 주십시오.

**Linux (Ubuntu/Debian):**
1. Cleona 웹사이트 또는 GitHub Releases에서 .deb 파일을 다운로드합니다.
2. 더블클릭하거나 터미널에서 다음 명령으로 설치합니다: `sudo dpkg -i cleona-chat_VERSION_amd64.deb`
3. 애플리케이션 메뉴 또는 터미널에서 `cleona-chat` 명령으로 Cleona를 실행합니다.

**Linux (Fedora/openSUSE):**
1. Cleona 웹사이트 또는 GitHub Releases에서 .rpm 파일을 다운로드합니다.
2. 다음 명령으로 설치합니다: `sudo dnf install cleona-chat-VERSION.x86_64.rpm`
3. 애플리케이션 메뉴 또는 터미널에서 `cleona-chat` 명령으로 Cleona를 실행합니다.

**Linux (모든 배포판 -- AppImage):**
1. Cleona 웹사이트 또는 GitHub Releases에서 .AppImage 파일을 다운로드합니다.
2. 파일을 실행 가능하게 만듭니다: 마우스 오른쪽 버튼 클릭 → 속성 → 실행
   가능, 또는 터미널에서: `chmod +x cleona-chat-VERSION-x86_64.AppImage`
3. 더블클릭하거나 터미널에서 다음 명령으로 실행합니다: `./cleona-chat-VERSION-x86_64.AppImage`

**Windows:**
1. Cleona 웹사이트 또는 GitHub Releases에서 설치 프로그램을 다운로드합니다.
2. 설치 파일을 실행하고 안내에 따릅니다.
3. 시작 메뉴 또는 바탕화면 바로가기를 통해 Cleona를 실행합니다.

### 아이덴티티 생성하기

처음 실행하면 Cleona가 자동으로 새로운 아이덴티티를 생성합니다. 표시
이름을 설정할 수 있으며, 이는 연락처에게 보이는 이름입니다. 이 이름은
언제든지 변경할 수 있습니다.

### 시드 문구 기록하기 -- 가장 중요한 단계

아이덴티티를 생성한 후 Cleona는 24개의 단어를 보여줍니다. 이것이 바로
여러분의 **시드 문구(Seed-Phrase)**이며, 개인 복구 키입니다.

**이 24개의 단어를 종이에 적어 안전한 곳에 보관하십시오.**

왜 이것이 그토록 중요할까요?

- 휴대폰이 고장 나거나 분실되거나 도난당하더라도, 이 24개의 단어만 있으면
  새 기기에서 아이덴티티 전체를 복구할 수 있습니다.
- 시드 문구가 없으면 되돌릴 방법이 없습니다. "비밀번호 찾기" 버튼도 없고,
  계정을 되찾아 줄 고객지원도 없습니다 -- 애초에 서버에 저장된 계정 자체가
  존재하지 않기 때문입니다.
- 시드 문구를 절대 다른 사람에게 알려주지 마십시오. 이 단어들을 아는
  사람은 여러분 행세를 할 수 있습니다.

나중에 다시 확인하고 싶다면, 설정의 "보안" 항목에서도 시드 문구를 찾을 수
있습니다.

### 첫 연락처 추가하기

누군가와 채팅하려면 먼저 그 사람을 연락처로 추가해야 합니다. 이를 위한
여러 방법이 있으며, 다음 섹션에서 모두 설명합니다.

## 3. 연락처

### QR 코드 스캔하기(권장)

연락처를 추가하는 가장 쉬운 방법입니다.

1. 상대방이 자신의 아이덴티티 상세 페이지를 열어(상단 바에서 자신의 이름을
   탭) QR 코드를 보여줍니다.
2. 플러스 버튼을 탭하고 "QR 코드 스캔"을 선택합니다.
3. 휴대폰을 상대방의 QR 코드 앞에 가져다 댑니다.
4. 연락처 요청이 자동으로 전송됩니다. 상대방이 이를 수락하면 서로 대화할
   수 있습니다.

직접 만나는 경우, QR 코드는 누구와 연락처를 교환하는지 정확히 알 수
있기 때문에 가장 안전한 방법입니다.

### NFC(휴대폰을 맞대기)

두 기기 모두 NFC를 지원하는 경우:

1. 양쪽 모두 연락처 추가 기능을 엽니다.
2. 두 휴대폰의 뒷면을 서로 맞댑니다.
3. 연락처 정보가 자동으로 교환됩니다.

NFC는 QR 코드와 마찬가지로 높은 보안성을 제공합니다. 두 사람이 물리적으로
나란히 있을 때만 교환이 이루어지기 때문입니다.

### 링크 공유하기(cleona:// URI)

연락처 링크를 이메일, SMS 또는 다른 메신저를 통해 전송할 수도 있습니다.

1. 자신의 아이덴티티 상세 페이지를 엽니다.
2. cleona:// 링크를 복사합니다.
3. 여러분을 추가하고자 하는 사람에게 링크를 전송합니다.
4. 상대방이 링크를 열거나, 연락처 추가 대화상자에 붙여넣습니다.

유의하십시오: 이 방법을 사용할 때는 전송 경로상에서 링크가 변조되지
않았음을 신뢰해야 합니다. 특히 민감한 연락처의 경우에는 QR 코드나 NFC
방식을 권장합니다.

### 연락처 요청 수락하기

누군가 연락처 요청을 보내면 받은함(하단 바의 마지막 탭)에 표시됩니다.
여기에서 다음을 할 수 있습니다.

- **수락** -- 해당 사용자가 연락처에 추가됩니다.
- **거절** -- 요청이 폐기됩니다.
- **차단** -- 해당 사용자는 더 이상 요청을 보낼 수 없습니다.

### 검증 단계

Cleona는 연락처의 아이덴티티가 얼마나 확실하게 확인되었는지를 보여줍니다.

| 단계 | 의미 |
|-------|-----------|
| 미확인 | Node-ID 또는 링크만 받은 상태입니다. |
| 확인됨 | 키 교환이 성공적으로 이루어져 암호화된 통신이 가능합니다. |
| 검증됨 | 직접 만나서 QR 코드나 NFC로 검증을 완료했습니다. |
| 신뢰함 | 이 연락처를 신뢰할 수 있다고 명시적으로 표시했습니다. |

단계가 높을수록 실제로 올바른 상대방과 대화하고 있다는 확신이 커집니다.

## 4. 메시지

### 텍스트 보내기와 받기

하단의 입력창에 메시지를 입력한 뒤 엔터 키나 전송 버튼을 누르기만 하면
됩니다. 메시지는 기기를 떠나기 전에 자동으로 암호화됩니다.

수신된 메시지는 채팅 기록에 표시됩니다. 체크 표시는 메시지가 전달되었는지를
알려줍니다.

### 사진, 동영상, 파일 보내기

여러 가지 방법이 있습니다.

- **입력창의 클립 아이콘**: 탭하여 갤러리나 파일 시스템에서 파일, 사진
  또는 동영상을 선택합니다.
- **드래그 앤 드롭**(데스크톱): 파일을 채팅 창으로 끌어다 놓기만 하면
  됩니다.
- **클립보드에서 붙여넣기**(데스크톱): 이미지를 복사한 뒤 채팅창에
  붙여넣습니다.

작은 파일(256KB 미만)은 바로 함께 전송됩니다. 더 큰 파일은 2단계 절차로
전송됩니다. 먼저 파일 전송이 예고된 뒤, 여러 부분으로 나뉘어 전달됩니다.

### 음성 메시지

1. 입력창의 마이크 버튼을 길게 누릅니다.
2. 메시지를 말합니다.
3. 버튼에서 손을 떼면 메시지가 전송됩니다.

기기에서 음성 인식이 활성화되어 있으면(설정 참조), 음성 메시지가
자동으로 텍스트로 변환됩니다. 상대방은 녹음본과 변환된 텍스트를 모두
보게 됩니다.

### 메시지에 답장하기(인용)

특정 메시지에 답장하려면:

1. 메시지 옆의 점 세 개 메뉴를 엽니다.
2. "답장"을 선택합니다.
3. 입력창 위에 인용된 메시지가 담긴 배너가 표시됩니다.
4. 답장을 작성하여 전송합니다.

인용된 메시지가 답장에 함께 표시되어 어떤 메시지에 대한 답변인지 명확히
알 수 있습니다.

### 메시지 수정 및 삭제

- **수정:** 메시지의 점 세 개 메뉴에서 "수정"을 선택합니다. 텍스트를
  변경한 뒤 다시 전송합니다. 상대방에게는 메시지가 수정되었다는 표시가
  보입니다. 수정은 전송 후 15분 이내에만 가능합니다.
- **삭제:** 메시지의 점 세 개 메뉴에서 "삭제"를 선택합니다. 메시지는
  나와 상대방 모두에게서 제거됩니다. 자신의 메시지는 언제든지 삭제할 수
  있으며, 삭제에는 시간 제한이 없습니다.

### 이모지 반응

답장을 작성하는 대신 이모지로 메시지에 반응할 수 있습니다.

1. 점 세 개 메뉴를 열거나 메시지를 길게 누릅니다.
2. 빠른 선택 목록에서 이모지를 고르거나, 전체 목록을 보려면 이모지
   선택기를 엽니다.
3. 반응이 메시지 아래에 표시됩니다.

### 텍스트 복사하기

메시지의 점 세 개 메뉴를 통해 메시지 텍스트를 클립보드에 복사할 수
있습니다.

### 메시지 검색하기

채팅 창 상단에서 검색 기능을 찾을 수 있습니다. 검색어를 입력하면 Cleona가
현재 채팅 내의 모든 검색 결과를 보여줍니다. 화살표 키를 사용해 검색 결과
사이를 이동할 수 있습니다.

시작 화면에는 모든 대화를 하나의 검색어로 검색할 수 있는, 탭을 아우르는
검색 필터도 별도로 마련되어 있습니다.

### 링크 미리보기

링크를 전송하면 Cleona가 자동으로 미리보기(제목, 설명, 미리보기 이미지)를
생성합니다. 이 미리보기는 여러분의 기기에서 생성되어 함께 전송되므로,
상대방은 링크된 웹사이트에 접속할 필요가 없습니다.

받은 링크를 탭하면 일반 브라우저로 열지, 시크릿 모드로 열지, 아니면 열지
않을지를 선택할 수 있습니다.

## 5. 그룹

### 그룹 만들기

1. "그룹" 탭으로 이동합니다.
2. 플러스 버튼을 탭합니다.
3. 그룹 이름을 입력합니다.
4. 초대하고자 하는 연락처를 선택합니다.
5. "만들기"를 탭합니다.

초대된 연락처는 알림을 받고 그룹에 참여할 수 있습니다.

### 멤버 초대하기

그룹을 만든 뒤에도 추가로 연락처를 초대할 수 있습니다.

1. 그룹 정보를 엽니다(그룹 목록의 점 세 개 메뉴 또는 그룹 채팅 상단
   바).
2. "초대"를 탭합니다.
3. 추가하고자 하는 연락처를 선택합니다.

### 역할

모든 그룹에는 세 가지 역할이 있습니다.

- **소유자(Owner):** 완전한 권한을 가집니다. 멤버를 추가 및 제거하고,
  관리자를 임명하며, 그룹을 관리할 수 있습니다. 소유자는 자신의 지위를
  다른 멤버에게 넘길 수도 있습니다.
- **관리자(Admin):** 멤버를 제거하고 관리 업무를 도울 수 있습니다.
- **멤버(Mitglied):** 메시지를 읽고 쓸 수 있습니다.

### 그룹 나가기

1. 그룹 목록의 점 세 개 메뉴를 엽니다.
2. "나가기"를 선택합니다.
3. 결정을 확인합니다.

그룹을 나가더라도 기존에 보낸 메시지는 다른 멤버들에게 계속 표시됩니다.

## 6. 공개 채널

### 채널이란 무엇인가요?

채널은 Cleona 네트워크 내의 공개 토론 게시판입니다. 그룹과 달리 채널은
초대받지 않아도 누구나 읽을 수 있습니다. 소유자와 관리자만 게시물을 올릴
수 있으며, 구독자는 이를 읽습니다.

### 채널 찾기 및 가입하기

1. "채널" 탭으로 이동합니다.
2. "검색" 탭을 엽니다.
3. 이름이나 주제로 사용 가능한 채널을 검색합니다.
4. 채널을 탭한 뒤 "구독"을 탭합니다.

채널은 언어별로 필터링할 수 있습니다. 일부 채널은 "청소년 이용 불가"로
표시되며, 이는 프로필에서 만 18세 이상임을 확인한 경우에만 표시됩니다.

### 나만의 채널 만들기

1. "채널" 탭으로 이동합니다.
2. 플러스 버튼을 탭합니다.
3. 채널 이름을 입력합니다(전체 네트워크에서 고유해야 합니다).
4. 언어를 선택하고 채널을 공개로 할지 비공개로 할지를 정합니다.
5. 선택 사항: 설명과 이미지를 추가합니다.
6. "만들기"를 탭합니다.

공개 채널의 경우 콘텐츠를 "청소년 이용 불가"로 분류할지 여부를 설정할
수 있습니다.

### 콘텐츠 신고하기

공개 채널에서 부적절한 콘텐츠를 발견하면 신고할 수 있습니다. Cleona는
탈중앙화된 모더레이션 시스템을 사용합니다. 신고는 네트워크에서 무작위로
선정된 멤버들이 평가합니다(일종의 "배심원단"과 같은 방식입니다). 위반
사실이 확인되면 해당 채널은 경고를 받습니다. 위반이 반복되면 검색
색인에서 순위가 낮아지거나 차단됩니다.

### 시스템 채널

Cleona에는 기본으로 내장된 두 개의 시스템 채널이 있습니다.

- **버그 로그(Bug Log):** Cleona가 오류를 감지하면 익명화된 오류 보고서를
  전송할지 물어봅니다. 이 보고서는 버그 로그 채널에 게시되어 커뮤니티가
  열람할 수 있습니다. 개인 정보는 전혀 전송되지 않으며 오직 기술적인 오류
  설명만 포함됩니다. 직접 로그 리포트를 수동으로 전송할 수도 있습니다
  (미리보기 대화상자와 명시적 동의 포함).
- **기능 요청(Feature Requests):** 사용자가 원하는 기능을 제안하고 기존
  제안에 투표할 수 있는 채널입니다. 제안은 득표수에 따라 정렬됩니다.

두 시스템 채널 모두 25MB의 용량 제한이 있으며, 배심원 모더레이션
시스템에 의해 관리됩니다.

## 7. 통화

### 음성 통화 시작하기

1. 통화하고자 하는 연락처와의 채팅을 엽니다.
2. 상단 바의 전화 아이콘을 탭합니다.
3. 상대방이 전화를 받을 때까지 기다립니다.

통화 중에는 통화 시간이 표시되며, 음소거와 스피커 기능을 사용할 수
있습니다.

통화를 종료하려면 빨간색 종료 버튼을 탭합니다.

### 영상 통화 시작하기

1. 연락처와의 채팅을 엽니다.
2. 상단 바의 카메라 아이콘을 탭합니다.
3. 여러분의 영상은 작은 창에 표시되고, 상대방의 영상은 큰 화면 영역에
   표시됩니다.

통화 중에 전면 카메라와 후면 카메라 사이를 전환할 수 있습니다.

### 수신 전화

누군가 전화를 걸면 발신자의 이름이 담긴 알림 창이 나타납니다. 다음 중
선택할 수 있습니다.

- **수락** -- 통화가 시작됩니다.
- **거절** -- 발신자에게 알림이 전달됩니다.

이미 통화 중인 경우 새로운 전화는 자동으로 거절됩니다.

### 그룹 통화

여러 사람이 동시에 참여하는 그룹 통화도 진행할 수 있습니다. 통화는
지능형 중계 트리를 통해 구성되어, 모든 참가자가 서로 직접 연결될 필요가
없습니다. 모든 통화는 처음부터 끝까지 암호화됩니다.

### 통화 암호화

모든 통화는 통화가 지속되는 동안에만 존재하는 일회용 키로 암호화됩니다.
통화가 종료되면 이 키는 즉시 삭제됩니다. 누구도 지나간 통화를 나중에
복호화할 수 없습니다.

## 8. 캘린더

Cleona에는 암호화되고 완전히 탈중앙화된 방식으로 작동하는 캘린더가
내장되어 있습니다 -- 클라우드 서비스 없이 말입니다.

### 보기 방식

캘린더는 일, 주, 월, 년, 할 일 목록의 다섯 가지 보기 방식을 제공합니다.
캘린더 화면 상단의 탭을 통해 전환할 수 있습니다.

### 일정 만들기

시간대를 탭하거나 추가 버튼을 사용해 새로운 일정을 만듭니다. 제목, 날짜,
시간, 장소, 메모를 입력할 수 있습니다. 일정은 기기에 암호화되어
저장됩니다.

### 반복 일정

일정은 매일, 매주, 매월 또는 매년 반복되도록 설정할 수 있습니다. 격주
화요일이나 매월 1일처럼 패턴을 조정할 수 있으며, 종료일이나 반복 횟수를
지정할 수도 있습니다.

### 연락처 초대하기

일정을 만들거나 수정할 때 Cleona 연락처를 초대할 수 있습니다.
초대받은 사람은 암호화된 캘린더 초대장을 받고 참석, 불참, 미정으로
응답할 수 있습니다. 일정이 변경되면 초대받은 모든 사람에게 자동으로
전달됩니다.

### 여유/예약 표시

일정의 세부 내용을 공개하지 않고도 연락처와 여유/예약 상태를 공유할 수
있습니다. 전체 세부 정보 공개, 시간대만 공개, 비공개의 세 가지 개인정보
보호 수준이 있습니다. 기본값을 설정한 뒤 연락처별로 개별 설정을 덮어쓸
수 있습니다.

### 알림

일정에는 시작 전 시스템 알림을 발생시키는 리마인더를 설정할 수
있습니다. 필요한 경우 알림을 다시 알림(스누즈) 처리할 수 있습니다.

### 외부 캘린더 동기화

Cleona는 외부 캘린더 서비스와 동기화할 수 있습니다.

- **CalDAV** -- Nextcloud, Radicale 등 CalDAV를 지원하는 모든 서버와
  연결할 수 있습니다.
- **Google 캘린더** -- 안전한 OAuth2 인증을 통한 Google Calendar API
  동기화입니다.
- **로컬 CalDAV 서버** -- Cleona는 기기에서 로컬 CalDAV 서버를 실행할
  수 있어, Thunderbird, Outlook, Apple 캘린더, Evolution 같은 데스크톱
  캘린더 앱이 Cleona 캘린더와 동기화될 수 있습니다.
- **Android 시스템 캘린더** -- Cleona의 일정을 Android 기기에 내장된
  캘린더 앱으로 전송할 수 있습니다.
- **ICS 파일** -- 표준 iCalendar 형식으로 일정을 가져오거나 내보낼 수
  있습니다.

### PDF 내보내기

모든 캘린더 보기(일, 주, 월, 년)를 PDF 문서로 인쇄하거나 내보낼 수
있습니다.

## 9. 설문조사

모든 채팅이나 그룹에서 의견을 수렴하거나 일정을 조율하기 위한
설문조사를 만들 수 있습니다.

### 설문 유형

Cleona는 다섯 가지 유형의 설문조사를 지원합니다.

- **단일 선택** -- 참가자가 하나의 항목을 선택합니다.
- **다중 선택** -- 참가자가 여러 항목을 선택할 수 있습니다.
- **일정 조율 설문** -- 모두에게 맞는 일정을 찾습니다. 각 참가자는
  일정을 가능함, 미정, 불가능으로 표시합니다.
- **척도** -- 숫자 척도(예: 1~5)로 무언가를 평가합니다.
- **자유 서술형** -- 참가자가 직접 답변을 작성합니다.

### 설문 만들기

채팅을 열고 설문 아이콘을 탭합니다(또는 첨부 메뉴를 이용합니다). 설문
유형을 선택하고 질문과 선택지를 작성한 뒤 설문을 전송합니다. 설문은
채팅 내에 메시지 형태로 표시됩니다.

### 투표하기

설문을 탭하여 투표합니다. 투표는 언제든지 변경하거나 철회할 수
있습니다.

### 익명 투표

설문은 익명 투표로 설정할 수 있습니다. 이 기능이 활성화되면 투표는
암호학적으로 익명 처리되어, 설문 작성자를 포함해 누구도 누가 무엇에
투표했는지 알 수 없습니다. 다만 득표수는 계속 표시됩니다.

### 일정 조율 설문을 캘린더로

일정 조율 설문이 마감되면, 탭 한 번으로 최종 선정된 일정을 캘린더
항목으로 바로 변환할 수 있습니다.

## 10. 다중 아이덴티티

### 왜 여러 개의 아이덴티티가 필요한가요?

직장 생활과 사생활을 분리하고 싶다고 상상해 보십시오 -- 마치 두 개의
서로 다른 전화번호를 쓰는 것과 비슷하지만, 두 번째 휴대폰은 필요
없습니다. Cleona에서는 하나의 기기에서 여러 개의 아이덴티티를 사용할
수 있습니다. 각 아이덴티티는 고유한 이름, 프로필 사진, 연락처, 대화
내역을 가집니다.

### 새 아이덴티티 만들기

1. 상단 바에 현재 아이덴티티가 탭 형태로 표시됩니다.
2. 아이덴티티 탭 오른쪽의 더하기 기호(+)를 탭합니다.
3. 새 아이덴티티의 이름을 입력합니다.
4. 완료 -- 새 아이덴티티가 곧바로 활성화됩니다.

### 아이덴티티 전환하기

상단 바의 아이덴티티 탭을 탭하기만 하면 됩니다. 전환은 즉시 이루어지며,
대기 시간이나 다시 불러오기가 필요 없습니다.

### 모든 아이덴티티가 동시에 작동합니다

중요한 점: 모든 아이덴티티는 동시에 활성 상태입니다. 현재 "직장용"으로
표시되어 있더라도 "개인용" 아이덴티티는 계속해서 메시지를 수신합니다.
어떤 아이덴티티를 선택해 두었든 아무것도 놓치지 않습니다.

### 아이덴티티 상세 페이지

현재 활성화된 아이덴티티의 탭을 탭하면 상세 페이지가 열립니다. 여기에서
다음을 할 수 있습니다.

- 연락처를 위한 QR 코드를 표시합니다.
- 프로필 사진을 변경하거나 삭제합니다.
- 프로필 소개글을 추가합니다.
- 표시 이름을 변경합니다.
- 이 아이덴티티에 사용할 디자인(스킨)을 선택합니다.
- 더 이상 필요하지 않은 아이덴티티를 삭제합니다.

### 아이덴티티 삭제하기

아이덴티티를 삭제하면 연락처들에게 이 사실이 통지됩니다. 해당
아이덴티티와 관련된 모든 데이터가 기기에서 제거됩니다. 이 작업은
되돌릴 수 없습니다.

---

## 11. 멀티 디바이스

### 여러 기기에서 Cleona 사용하기

동일한 아이덴티티를 최대 5개의 기기에서 동시에 사용할 수 있습니다. 한
기기가 기본(primary) 기기가 되어 시드 문구를 보유하며, 다른 기기들은
이 기본 기기와 연결됩니다.

### 새 기기 연결하기

1. 기본 기기에서 설정을 엽니다.
2. "연결된 기기"로 이동합니다.
3. "새 기기 연결"을 선택합니다.
4. 새 기기에 Cleona를 설치하고, 처음 실행할 때 "기존 기기와 연결"을
   선택합니다.
5. 기본 기기에 표시되는 페어링 QR 코드를 스캔하거나 페어링 링크를
   사용합니다.

연결된 기기는 기본 기기로부터 위임 인증서를 받습니다. 연결된 기기에서
전송된 메시지는 위임된 키로 암호학적으로 서명되어, 연락처가 해당
메시지가 실제로 여러분의 아이덴티티에서 온 것임을 확인할 수 있습니다.

### 작동 방식

- 기본 기기가 시드 문구와 마스터 키를 보유합니다.
- 연결된 기기는 파생된 서명 키와 위임 인증서를 받습니다 -- 시드 문구
  자체를 받는 일은 결코 없습니다.
- 모든 기기는 동일한 아이덴티티와 연락처를 공유합니다. 메시지는 모든
  기기에 도착합니다.
- 위임 인증서는 만료되기 전에 자동으로 갱신됩니다.

### 기기 관리

설정을 열고 "연결된 기기"로 이동하면 연결된 모든 기기와 각 기기의
상태, 최근 활동을 확인할 수 있습니다. 기기를 분실하거나 도난당한 경우
언제든지 해당 기기의 연결을 해지할 수 있습니다.

### 비상 키 교체

기기가 침해당했다고 의심되는 경우 비상 키 교체를 실행할 수 있습니다.
이 과정에서 새로운 키가 생성되며, 교체는 나머지 기기들 중 과반수의
승인을 받아야 완료됩니다. 이를 통해 도난당한 기기 하나만으로 임의로
키를 교체하는 것을 방지합니다.

## 12. 복구

### 시드 문구 사용하기

기기를 분실했거나 새 기기를 설정하는 경우:

1. 새 기기에 Cleona를 설치합니다.
2. 처음 실행할 때 "복구"를 선택합니다.
3. 24개의 단어를 입력합니다.
4. Cleona가 아이덴티티를 복구하고 자동으로 기존 연락처들에게 연락합니다.
5. 연락처들이 여러분의 연락처 정보, 그룹 멤버십, 메시지 기록으로
   응답합니다.

복구는 세 단계로 이루어집니다.
- 먼저 연락처와 그룹이 복구됩니다.
- 그다음 각 대화의 최근 50개 메시지가 복구됩니다.
- 마지막으로 전체 메시지 기록이 복구됩니다.

복구가 이루어지려면 연락처 중 단 한 명만 온라인 상태여도 충분합니다.

### 가디언 복구(신뢰할 수 있는 사람들)

최대 5명의 신뢰할 수 있는 사람을 "가디언(Guardian)"으로 지정할 수
있습니다. 이때 복구 키가 5개 조각으로 나뉘어 각 가디언이 하나씩 받게
됩니다. 아이덴티티를 복구하려면 5개 조각 중 3개만 있으면 충분합니다.

즉, 시드 문구를 잃어버렸다 하더라도 가디언 중 3명이 힘을 합치면 계정을
복구할 수 있다는 뜻입니다. 어떤 가디언도 혼자서는 여러분의 데이터에
접근할 수 없으며, 항상 최소 3명이 필요합니다.

가디언을 설정하는 방법:
1. 설정을 엽니다.
2. "보안"으로 이동합니다.
3. "가디언 복구"를 선택합니다.
4. 신뢰할 수 있는 연락처 5명을 선택합니다.

### 연락처가 곧 여러분의 백업인 이유

기존 메신저에서는 여러분의 데이터가 제공업체의 서버에 저장됩니다.
Cleona에는 서버가 없지만, 대신 연락처들이 그 역할을 담당합니다.
메시지를 보내면 수신자가 오프라인일 경우를 대비해 공통 연락처들이
암호화된 사본을 저장해 둡니다. 복구 시에는 연락처들이 여러분에게
데이터를 돌려줍니다.

즉, 활동 중인 연락처가 많을수록 백업의 신뢰성이 높아집니다. 정기적으로
온라인 상태인 연락처 한 명만 있어도 성공적인 복구에는 충분합니다.

## 13. 설정

설정은 우측 상단의 톱니바퀴 아이콘을 통해 접근할 수 있습니다.

### 알림 및 벨소리

- 수신 전화를 위한 여섯 가지 벨소리 중에서 선택할 수 있습니다.
- 메시지 알림음을 설정할 수 있습니다.
- Android 기기에서는 추가로 진동을 켜거나 끌 수 있습니다.

### 디자인(스킨)

Cleona는 Teal, Ocean, Sunset, Forest, Amethyst, Fire, Storm, Slate,
Gold, Contrast의 열 가지 디자인을 제공합니다. Contrast 디자인은 최고
수준의 접근성 기준(WCAG AAA)을 충족하며, 시력이 제한된 경우에도 특히
읽기 쉽습니다.

각 아이덴티티는 고유한 디자인을 가질 수 있습니다. 디자인은 아이덴티티
상세 페이지(활성 아이덴티티 탭을 탭)에서 변경할 수 있습니다.

또한 설정의 "화면 모드" 항목에서 밝은 테마, 어두운 테마, 시스템 테마
중에서 선택할 수 있습니다.

### 언어 변경하기

Cleona는 33개 언어로 제공되며, 아랍어나 히브리어처럼 오른쪽에서
왼쪽으로 쓰는 언어도 포함됩니다. 설정의 "언어" 항목에서 언어를 변경할
수 있습니다.

### 저장 용량 제한

Cleona가 기기에서 사용할 수 있는 저장 용량(100MB에서 2GB 사이)을
설정할 수 있습니다. 한도에 도달하면 오래된 미디어가 자동으로
이동되거나 삭제됩니다 -- 텍스트 메시지는 항상 보존됩니다.

### 미디어 아카이빙

집에 네트워크 저장 장치(NAS)나 공유 폴더가 있다면, Cleona가 미디어를
자동으로 그곳에 이동시킬 수 있습니다. SMB, SFTP, FTPS, WebDAV를
지원합니다.

단계별 저장 방식은 다음과 같이 작동합니다.
- 처음 30일: 모든 것이 기기에 그대로 남아 있습니다.
- 30일 후: 미리보기 이미지만 기기에 남고, 원본은 아카이브로
  이동합니다.
- 90일 후: 작은 미리보기 이미지만 기기에 남습니다.
- 1년 후: 자리표시자(placeholder)만 남고, 원본은 아카이브에 안전하게
  보관됩니다.

아카이브된 미디어는 언제든지 탭하여 다시 불러올 수 있습니다 -- 단, 홈
네트워크에 연결되어 있어야 합니다. 특히 중요한 미디어는 고정하여 절대
이동되지 않도록 할 수 있습니다.

### 음성 메시지 텍스트 변환

이 기능을 활성화하면 음성 메시지가 기기에서 로컬로 텍스트로
변환됩니다(오픈소스 모델 Whisper 사용). 변환된 텍스트는 녹음본과 함께
상대방에게 전송됩니다. 변환은 전적으로 기기 내에서 이루어지며, 외부
서비스로는 어떤 데이터도 전송되지 않습니다.

### 자동 다운로드

미디어를 자동으로 다운로드할 크기 기준을 설정할 수 있습니다. 예를 들어
이미지는 자동으로 불러오되, 용량이 큰 동영상은 직접 결정하도록 설정할
수 있습니다.

### 연결된 기기

설정의 이 영역에서 연결된 기기를 관리할 수 있습니다. 자세한 내용은
멀티 디바이스 챕터를 참고하십시오.

## 14. 보안

### 포스트 퀀텀 암호화란 무엇인가요?

오늘날의 암호화는 일반 컴퓨터로는 풀기 극도로 어려운 수학적 문제를
기반으로 합니다. 양자 컴퓨터는 미래에 이러한 문제 중 일부를 빠르게 풀
수 있을 것으로 예상됩니다. 포스트 퀀텀 암호화는 양자 컴퓨터도 견뎌낼
수 있는 추가적인 방식을 사용합니다.

Cleona는 두 가지 접근 방식을 결합합니다. 신뢰성을 위한 고전적
암호화와 미래 대비를 위한 포스트 퀀텀 방식입니다. 이를 통해 현재와
미래의 위협 모두로부터 동시에 보호받을 수 있습니다.

각각의 메시지마다 고유한 키가 생성됩니다. 설령 공격자가 한 메시지의
키를 해독하더라도, 그 키로는 다른 어떤 메시지도 읽을 수 없습니다.

### 서버가 없는 것이 왜 더 안전한가

기존 메신저에서는 메시지가 제공업체의 서버를 거쳐 전달됩니다. 그곳에서
암호화되어 있다 하더라도, 제공업체는 메타데이터(누가 언제 누구와
얼마나 자주, 어디서 통신하는지)에 접근할 수 있으며, 경우에 따라 법원의
명령에 따라 이를 제출해야 할 수도 있습니다.

Cleona에는 그러한 중앙 집중점이 존재하지 않습니다. 메시지는 기기에서
기기로 직접 이동합니다. 모든 메타데이터가 한곳에 모이는 장소 자체가
없습니다. 누구도 단 하나의 데이터 포인트만으로 여러분의 통신 패턴을
재구성할 수 없습니다.

### 오프라인일 때는 어떻게 되나요?

메시지를 보냈는데 수신자가 오프라인인 경우:

1. Cleona는 먼저 메시지를 직접 전달하려고 시도합니다.
2. 이것이 실패하면 공통 연락처를 거쳐 전달됩니다.
3. 동시에 메시지는 암호화된 조각으로 나뉘어 네트워크의 여러 노드에
   분산 저장됩니다(10개 조각으로 이루어진 퍼즐에서 7개만 있으면 전체
   그림을 완성할 수 있는 것과 비슷합니다).
4. 메시지는 최대 7일간 보관됩니다.

수신자가 다시 온라인이 되는 즉시 메시지가 전달됩니다. 메시지가
도착하면 확인 알림을 받게 됩니다.

### 검열 방지

네트워크가 기본 연결 방식(UDP)을 차단하는 경우, Cleona는 자동으로
탐지와 차단이 더 어려운 대체 전송 방식(TLS)으로 전환합니다. 이 과정은
투명하게 이루어지며, 여러분이 별도로 설정할 필요는 없습니다.

### 안전한 키 저장

지원되는 플랫폼에서는 Cleona가 운영체제의 안전한 키 저장소(Android
Keystore, iOS Keychain, macOS Keychain)에 암호화 키를 저장합니다.
이용 가능한 경우 하드웨어 기반의 키 보호 기능도 제공됩니다.

### 데이터베이스 암호화

모든 메시지, 연락처, 설정은 기기에 암호화되어 저장됩니다. 누군가 파일
시스템에 접근하더라도 암호화 키가 없으면 아무것도 읽을 수 없습니다.
이 키는 여러분의 아이덴티티로부터 파생되며 오직 기기에만 존재합니다.

### 폐쇄형 네트워크

Cleona는 폐쇄형 네트워크로 작동합니다. 모든 네트워크 패킷은 인증되어
있어 정당한 Cleona 기기만 참여할 수 있습니다. 이를 통해 외부인이
위조된 메시지를 주입하거나 네트워크 트래픽을 도청하는 것을 방지합니다.

## 15. 소프트웨어 업데이트

### 업데이트는 어떻게 받나요?

Cleona는 여러 경로를 통해 업데이트할 수 있습니다. 목표는 특정 배포
경로가 장애를 일으키거나 차단되더라도 계속 업데이트를 받을 수 있도록
하는 것입니다.

1. **App Store / Play Store:** Cleona를 앱 스토어를 통해 설치했다면,
   평소처럼 스토어를 통해 업데이트를 받습니다.
2. **GitHub Releases:** 프로젝트의 GitHub 페이지에서 모든 플랫폼용
   서명된 설치 패키지를 찾을 수 있습니다.
3. **네트워크 내 업데이트(In-Network-Update):** 여러분의 네트워크에
   있는 다른 Cleona 사용자가 이미 최신 버전을 가지고 있다면, Cleona는
   외부 서버 없이 P2P 네트워크를 통해 직접 업데이트를 받아올 수
   있습니다. 이때 새 버전은 오류 정정 조각으로 분할되어 여러 노드에
   분산됩니다. 여러분의 기기는 충분한 조각을 모아 업데이트를
   재구성합니다. 진위 여부는 개발자의 Ed25519 서명으로 검증됩니다.
4. **초대 링크:** 새로운 사용자가 Cleona를 설치하고 네트워크에
   연결하는 데 필요한 모든 것을 담은 초대 링크를 만들 수 있습니다.
5. **물리적 전달:** 인터넷이 없는 환경에서는 USB 드라이브나 로컬
   네트워크를 통해 다른 사람에게 Cleona를 전달할 수 있습니다.

### 업데이트 알림

새로운 업데이트가 있으면 Cleona가 시작 화면에 알림을 표시합니다.
업데이트가 네트워크를 통해서도 이용 가능한 경우(네트워크 내 업데이트),
네트워크에서 직접 다운로드할 것인지 선택할 수 있습니다.

### 바이너리 배포

기본적으로 여러분의 기기는 네트워크상의 다른 사용자에게 업데이트를
전달하는 데 도움을 줍니다. 원하지 않는 경우 설정의 "네트워크" 항목에서
이 기능을 비활성화할 수 있습니다. 업데이트 조각을 위한 저장 공간
사용량은 제한되어 있으며(모바일 기기 5MB, 데스크톱 기기 20MB),
정기적으로 정리됩니다.

### 서명 검증

모든 업데이트는 암호학적으로 서명됩니다. Cleona는 업데이트를 설치하기
전에 자동으로 서명을 검증합니다. 이를 통해 P2P 네트워크를 통해 받은
업데이트라 하더라도 공식 개발자의 업데이트만 승인되도록 보장합니다.

## 16. 자주 묻는 질문

### "인터넷 없이 Cleona를 사용할 수 있나요?"

아니요, Cleona는 메시지를 주고받기 위해 네트워크 연결이 필요합니다.
다만 상대방과 동시에 온라인 상태일 필요는 없습니다. 수신자가 오프라인인
동안 보낸 메시지는 임시로 저장되었다가 양쪽이 다시 연결되는 즉시
자동으로 전달됩니다. 로컬 네트워크(예: 같은 WLAN)에서는 인터넷 연결
없이도 서로 통신할 수 있습니다.

### "시드 문구를 잃어버리면 어떻게 되나요?"

가디언을 설정해 두었다면, 5명 중 3명이 힘을 합쳐 접근 권한을 복구할 수
있습니다. 가디언도 시드 문구도 없다면, 안타깝게도 아이덴티티를 되찾을
방법이 없습니다. 그렇기 때문에 24개의 단어를 안전하게 보관하는 것이
매우 중요합니다.

### "누군가 제 메시지를 몰래 볼 수 있나요?"

아니요. 모든 메시지는 오직 그 메시지 하나에만 적용되는 일회용 키로
암호화됩니다. 오직 여러분과 상대방만이 메시지를 복호화할 수 있습니다.
중앙 서버도, 마스터 키도, 개발자를 위한 접근 권한도 존재하지
않습니다. 전송 경로상에서 메시지를 중계하는 기기라 하더라도 암호화된
무의미한 데이터만 볼 수 있습니다.

### "왜 전화번호가 필요 없나요?"

여러분의 아이덴티티가 순수하게 암호학적이기 때문입니다. 실명과
연결된 전화번호나 이메일 주소 대신, 기기에서 생성된 키 쌍이 여러분을
식별합니다. 연락처는 전화번호부가 아니라 QR 코드, NFC, 링크를 통해
추가합니다. 이는 메신저 계정이 실제 신원과 결부되지 않기 때문에 더
높은 프라이버시를 의미합니다.

### "Cleona에서 사람들을 어떻게 찾나요?"

Cleona는 전화번호나 이름으로 연락처를 검색하는 기능을 의도적으로
제공하지 않습니다 -- 이는 프라이버시 문제를 일으킬 수 있기
때문입니다. 대신 QR 코드, NFC, cleona:// 링크, 또는 공개 채널을 통해
직접 연락처 정보를 교환합니다. 이는 전화번호부를 찾아보는 대신
명함을 교환하는 것과 같습니다.

### "Cleona는 해외에서도 작동하나요?"

예. 인터넷 연결만 있다면 Cleona는 전 세계 어디에서나 작동합니다.
중앙 서버가 없기 때문에 특정 국가를 대상으로 서비스를 차단할 수도
없습니다. 또한 Cleona는 검열 방지 대체 수단을 갖추고 있어, 일반 연결
방식(UDP)이 차단되면 탐지와 차단이 더 어려운 대체 전송 방식(TLS)으로
자동 전환합니다.

### "Cleona는 무료인가요?"

예. Cleona는 무료이며 광고 없이 이용할 수 있습니다. 중앙 서버가
없기 때문에 운영을 위한 서버 비용도 발생하지 않습니다. 앱 내의
"후원" 메뉴에서 자발적으로 개발을 지원할 수 있는 방법을 찾을 수
있습니다.

### "메시지에 시계 아이콘이 표시됩니다 -- 무슨 의미인가요?"

이는 메시지가 아직 전달되지 않았음을 의미합니다. 상대방이 현재
오프라인 상태일 가능성이 높습니다. 메시지가 전달되면 아이콘이
바뀝니다. 메시지는 전달을 위해 최대 7일간 보관됩니다.

### "WhatsApp에서 Cleona로 옮겨갈 수 있나요?"

예, 다만 WhatsApp 채팅 내역은 옮길 수 없습니다. Cleona와 WhatsApp은
근본적으로 다른 시스템입니다. Cleona에서는 연락처를 하나씩 추가해야
합니다. 가장 간단한 방법은 WhatsApp 그룹에 cleona:// 링크를 게시하고
다른 사람들에게 그 링크로 여러분을 추가해 달라고 요청하는 것입니다.

### "여러 기기에서 동시에 Cleona를 사용할 수 있나요?"

예. 최대 5개의 기기를 동일한 아이덴티티로 연결할 수 있습니다. 한
기기가 기본 기기가 되어 시드 문구를 보유하며, 다른 기기들은 안전한
페어링 절차를 통해 연결됩니다. 모든 기기는 동일한 아이덴티티, 연락처,
대화 내역을 공유합니다. 자세한 내용은 멀티 디바이스 챕터를
참고하십시오.

### "App Store가 차단된 경우 업데이트는 어떻게 받나요?"

Cleona는 App Store나 웹사이트, 다운로드 서버에 의존하지 않고도 P2P
네트워크를 통해 직접 업데이트를 받을 수 있습니다. 네트워크에 있는
다른 사용자가 최신 버전을 가지고 있다면, 여러분의 기기는 그곳에서
업데이트를 받아올 수 있습니다. 진위 여부는 개발자의 디지털 서명으로
검증됩니다. 또는 연락처가 초대 링크나 USB 드라이브를 통해 앱을
전달해 줄 수도 있습니다. 자세한 내용은 "소프트웨어 업데이트" 챕터를
참고하십시오.
